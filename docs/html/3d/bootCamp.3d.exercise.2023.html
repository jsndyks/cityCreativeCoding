Task : 3D face

In this task, you will construct a 3D face using spheres and boxes.
You will then rotate the face and have its eyes change colour when the mouse is clicked.

  1. Using boxes and spheres, construct a face, consisting of two spheres for the eyes and elongated boxes for the nose and mouth. For this task, it might be helpful to think about the coordinate system in which the shapes are placed. Recommendation: place one eye at (-100, -100, 0) and the other at (100, -100, 0). For the nose, draw a box centred at (0,0,0) and elongated in the vertical (y) direction. For the mouth draw a box stretched along the horizontal (x) direction and translated along the positive y axis.
  2. Have the face rotate about the y axis, based on horizontal mouse movement.
  3. Change the eye colour to blue when the user presses the mouse.
  1. How might you replace the elongated box used for the mouth to show a smile?


Challenge : Box City

In this task, you will construct 2D array of boxes that resembles a skyline. You will then animate the boxes using Perlin noise.

  1. Copy the following code into a new Processing sketch.

{
  size(640, 480, P3D);

  background(0);
  stroke(255, 255, 0);
  fill(200, 200, 0);
  lights();

  translate(width/2, height/2, 0);
  rotateX(radians(45));
  rotateZ(radians(-mouseX));
  translate(-200, -200, 0);

  float d = 100;

  // Insert your code here

}
  2. Below the comment that says 'Insert your code here', create a doubly-nested forloop over x and y. For each loop, initialise the integer variable to 0, set the condition to the variable being less than 400, and for the increment, advance the variable by 20. We'll be rendering boxes on a grid that is 20 x 20 units.
  3. Inside the inner loop, make a matching call to pushMatrix() and popMatrix().
  4. After the call to pushMatrix(), draw a box that has width of 15 units, a height of 15 units, and depth of d units, where d is given in the code.
  5. Run the program. You should see a single box as shown below.

    What is happening is all the boxes are being rendered at the same place, then rotated based on mouse movement. How can you move the boxes so they don't overlap to achieve the result below?
  6. In the innermost loop, use Perlin noise method to change the value of d of each box, based on the variables x and y. You may find it helpful to multiply the values of x and y by a small number, like 0.01. Also, multiply the output of the Perlin noise value by a number like 200, to make the box taller. Run your program to check it produces results similar to those shown below (the random numbers will be different each time, so the shape of the box city will not exactly match that shown here):
  7. Now try setting d = 1000*pow(noise(0.01*x, 0.01*y), 5);. Notice the effect. What is this code doing?
  8. Using the elapsed time (measured by millis()), pass a third argument in to the noise method. This should create an animation. You may want to multiply the value returned by millis()by a small number, like 0.001.
  1. Try enhancing your city by adding texture images to the building boxes. Think creatively and experiment with developing your city model in other ways. Have fun trying things out!
