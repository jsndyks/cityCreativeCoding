Why not play with the alphabet and create your own characters?
Here is one way and one solution - but surely you can do something more interesting?

This Processing sketch that shows you one way to do this.

  • crazyAlphabet.pde

This example uses a StringDict, which is worth a look. It allows you to convert from one string (such as a character or letter in this case) to another (such as a set of coordinates through which the letter might be represented on screen).

So, for example the letter A might be represented by the coordinate pairs 0,100 50,0 50,0 100,100 25,50 75,50, whereas the letter B is a little more complex and may need more points - 0,0 80,0 80,0 100,10 100,10 100,30 100,30 80,40 80,40 0,40 80,40 100,50 100,50 100,80 100,80 80,100 80,100 0,100 0,100 0,0.


// -- create a StringDict to relate one string with another: 
StringDict font = new StringDict();

// -- relate the String "A" with a String containing a set of coordinates, same for "B", etc.
font.set("A", "0,100 50,0 50,0 100,100 25,50 75,50");  
font.set("B", "0,0 80,0 80,0 100,10 100,10 100,30 100,30 80,40 80,40 0,40 80,40 100,50
  100,50 100,80 100,80 80,100 80,100 0,100 0,100 0,0");

// -- etc!

If you do this for multiple letters, well you can create a graphical representation for the whole alphabet.
