Programming BootCamp Project Ideas: Steganography

This project involves generating hidden messages through steganography. That is, the art of concealing some text or image in plain sight.

If you want to send a secret message to someone you can encrypt it with some kind of code or cypher. But one of the problems with encryption is that it is usually obvious that a message has been encrypted and so can attract attention of others wanting to decrypt it. Alternatively you can hide a message in such a way that no one but the intended recipient of the message even knows there is a message to find. Hiding such messages is known as steganography.

For this project, you can think creatively about how you might use Processing to create some digital object such as some text, an image or some sound that hides within it some other message. Here are a couple of examples to consider, but you may be able to come up with others.

Text Steganography

In the Data Visualization session you learned how to read a text file into Processing. You could read a large text such as War of the Worlds from Project Gutenberg and select sentences from it to hide a message of your choice. For example, look at the last word of each sentence in the middle paragraph below to uncover a hidden message. 

“Now as I stood and examined it, finding a pleasure in the mere touch
of the contrivance, the thing I had expected happened. The bronze
panels suddenly slid up and struck the frame with a clang. I was in the
dark—trapped. So the Morlocks thought. At that I chuckled gleefully.

“I feel assured it’s this business of the Time Machine,” I said, and
took up the Psychologist’s account of our previous meeting. “It sounds
plausible enough tonight,” said the Medical Man; “but wait until tomorrow. 
I can’t argue this night”. I looked  into the thickness of the wood and 
thought of what it might hide. There were numbers of guns, pistols, and 
rifles. I felt assured that the Time Machine was  only to be recovered
by boldly penetrating these mysteries of underground.

“I could already hear their murmuring laughter as they came towards me.
Very calmly I tried to strike the match. I had only to fix on the
levers and depart then like a ghost. But I had overlooked one little
thing. The matches were of that abominable kind that light only on the
box.

You should be able to use Processing's String methods such as split() and startsWith() to extract and search for words in the text. This should allow you to select parts of the text that contain words of your choice.

Image Steganography

Can you tell the difference between these two rectangles?

Although they look the same, the first is coloured with fill(200, 100, 50) and the second with fill(201, 99, 51). We can use such small differences in colour value to hide other information. The data are stored digitally, but we cannot perceive the hidden data in the image pixel colours.

We can even store entire hidden images. The image below left contains a hidden image by varying the transparency of each image pixel between 240 and 255 – allowing 16 levels of grey to be stored in each pixel undetected. Because the transparency does not vary very much (between 240 and 255) it is not detectable when viewing casually. Yet if you extract those transparency values (using alpha()) and use them to determine a grey level between 0-255, you get the image below right.

For your project you could create a sketch that will hide one image inside another using the least important parts of pixel colour values so they are undetectable when displayed. You can then write an 'image decoder' that extracts and displays the hidden image. Here are a few hints to help you get started:

  • Make sure you have read through the Image Processing bootcamp lecture and have had some practice at manipulating images.
  • You can load images from files with loadImage()
  • You can create new images in processing with createImage() and you can set a new image to be the same dimensions as an existing one with PImage newImg = createImage(oldImg.width, oldImg.height, ARGB);
  • If you wish to read the pixel values of an image you must fist call loadPixels() before you use the pixels array.
  • If you wish to change the pixel values of an image you must call updatePixels() after you have changed the image's pixels array.
  • You can extract the red, green, blue, brightness and transparency values of a pixel with red(), green(), blue(), brightness() and alpha() respectively.
  • You can set the ith pixel value in the pixels array with pixels[i] = color(XXX) where XXX is any colour description such as (200,100,50).

To make this project more fun you might wish to find a partner and give each other a steganographic object (text or image) and challenge each other to try to find the secret data embedded within it. This will give you practice at both encoding and decoding messages.
