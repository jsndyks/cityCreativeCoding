A sequencer is a device for creating musical 'loops' by triggering sounds at regular intervals. Hardware sequencers first became widely available in the 1980s, being particularly popular for rhythm and bass sequencing. You can take advantage of Processing's ability to keep track of time to create your own software sequencer by triggering samples based on frame count or with Processing's millis() method.



To start you off, here is a basic sketch for keeping track of a 4-bar sequence of 16 notes. Instead of printing out the tick value you can trigger one or more samples using the play() of a loaded sample. 

import processing.sound.*;   // Import the library that does the sound handling.

int startTime;   // Time when sketch starts.
int bpm;         // Speed of sequence in beats per minute
int tick;        // Current 1/16th note (semiquaver) being played
int measures;    // Number of beats in repeating sequence

void setup()
{
  bpm = 60;
  measures = 16;
  tick = -1;
  startTime = millis();  // Compare current time with this to find time elapsed.
}

void draw()
{
  int elapsed = ((int)(bpm*(millis()-startTime)/15000))%measures;
  if (elapsed != tick)
  {
    tick = elapsed%measures;
    println(tick);
  }
}

Project Ideas

  • Find some public domain samples and build a drum machine using percussion sounds. Use interaction to allow the operator to create their own rhythms.
  • Generate sequences programmatically, perhaps with some randomness.
  • Use the sound library to generate synthesised sounds, perhaps like the influential Roland TB-303 bass sequencer.
  • Experiment with polyrhythms to create interesting evolving rhythmic patterns.
  • Combine some sequence with visuals, perhaps that are triggered by the sounds in your sequence.
