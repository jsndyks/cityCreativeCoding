Generative art involves constructing artwork from sets of relatively simple rules that allow the work to build itself as a computer program is run. Agent-based generative art places those rules inside many agents that interact with their environment to produce visual patterns.

For generative art projects you will create one or more classes to contain simple rules for describing an agent's behaviour. For examples, have a look at the bouncing ball and randomness pages to see how classes can be created and used to control the visual appearance of a sketch.

As a general guide, start by creating a simple class representing an agent. It is likely that as a minimum, your agent will contain two variables representing its screen position (x and y) as well as some rules that change its location. Next create a simple sketch that draws on the screen using this single agent. Then try adding several agents by creating an array of them. You should experiment with changes to both the visual appearance of your drawing (colour, line and fill styles etc.) as well as the rules that define your agent's behaviour.

How you design your agent is up to you. The key is to experiment to come up with something you like. If you are stuck for ideas, here are a few to get you started:

  • Create agents that respond to movement of the mouse. Allow them to leave 'trails' so they build up your picture as they move around.
  • Use your agents to provide the vertices of some simple shapes (e.g. three agents could represent the three corners of a triangle).
  • Create agents that interact with each other in some way (see the bouncing ball page for an example).
  • Create agents that have rules for changing their size and colour as well as position. Experiment with different visual appearances.
