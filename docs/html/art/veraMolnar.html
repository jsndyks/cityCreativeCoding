Vera Moln치r - (Des) Ordres

Vera Moln치r began working with computers in 1968 and established herself as pioneer of algorithmic art by programming pen plotters in Basic and Fortran.
She died in late 2023 a few weeks before her 100th birthday. This short obituary from Artsy is one of many that confirms and celebrates her vital role in "exploring the relationship between technology and art".

Her work is showcased in the Digital Art Museum, and I really enjoyed an exhibition hosted at Schloss Dagstuhl in the summer of 2022.

I find the Des Ordres - "some order" and perhaps "disorder" - exhibit particularly compelling.

The Digital Art Museum describes these pieces as involving ...

a pattern of concentric squares which is randomly disrupted in order to highlight the contrast between order and disorder and create tensions in the orthogonal structure, as if the squares were subject to a vibrating force

Let's see whether we can recreate some of this ordered disorder using our Processing skills...
and whether we learn anything about order, disorder, aesthetic, what Vera was doing and perhaps Processing in the process.

1. Draw a Square with quad()

Use the quad() method to do so - it takes eight parameters that specify four coordinate pairs (x,y), one for each corner, clockwise from top left.

Your quad should be square, centred on xPos,yPos and sSize by sSize in dimensions.

Use this code as the basis of your sketch, and fill in the eight arguments to quad() that will complete the task.

2. Create a Method to Draw a vQuad

Use quad() in your new vQuad() method, which should take three arguments, as shown below to specify the position and size of the square.

It should, once again create a square, centred on xPos,yPos and be sSize by sSize in dimensions.

            void vQuad (float x, float y, float s) {

              float sHalf = s/2;
            
              quad(
                x - sHalf, y - sHalf,
                x + sHalf, y - sHalf,
                x + sHalf, y + sHalf,
                x - sHalf, y + sHalf
              );
            }

3. Give the vQuad some Randomness

Adapt your vQuad() method to take a 4th parameter that adds some randomness to the quad coordinates.

Your quad() will no longer be square, as each coordinate should vary randomly by an amount up to the number given as the 4th argument.

If your 4th argument, the degree of randomness, is r, then adding random(-r, r) to each of the 8 coordinates will achieve this effect.

      void setup()
      {
        size(600, 600);
      }
      
      void draw()
      {
        background(255);
        stroke(32);
        noFill();
      
        float xPos   = 100;
        float yPos   = 100;
        float sSize  = 100;
      
        vQuad(xPos, yPos, sSize, 5);
      }

            void vQuad (float x, float y, float s, float r) {

              float sHalf = s/2;
            
              quad(
                x - sHalf + random(-r, r), y - sHalf + random(-r, r),
                x + sHalf + random(-r, r), y - sHalf + random(-r, r),
                x + sHalf + random(-r, r), y + sHalf + random(-r, r),
                x - sHalf + random(-r, r), y + sHalf + random(-r, r)
              );
            }

4. Create a Method to Draw vQuads within a vQuad

Now we need a new method vQuads that calls the vQuad method multiple times, each time creating a vQuad in the same position, but of a decreasing size.

So vQuads will need to take five parameters that in turn specify:
the central position in x and y, the size (of the outer vQuad), the randomness to use and the number of vQuads to be drawn.

      void setup()
      {
        size(600, 600);
      }
      
      void draw()
      {
        background(255);
        stroke(32);
        noFill();
        randomSeed(37196299); // <-- any number in here keeps random numbers the same in each draw() loop
      
        float xPos   = 200;
        float yPos   = 200;
        float sSize  = 200;
      
        vQuads(xPos, yPos, sSize, 5, 6); // <-- this method will need to call vQuad multiple times according to the 5th argument
      }

      void vQuads (float x, float y, float s, float r, int n) {

        for (int i=1; i<=n; i++) {
          vQuad(x, y, s*i/n, r);
        }
      }

5. Randomly Omit Some of the Quads

Add a 6th argument to vQuads that is the likelihood that each individual vQuad will be drawn.

This should vary from 0-1
A value of 1 will mean that every vQuad is drawn, and have no effect. A value of 0.1 will result in very sparse vQuads with only one in ten calls to vQuad resulting in a quad() being drawn.

Here is the code that calls vQuads, with the new parameter.

void setup()
{
  size(600, 600);
}

void draw()
{
  background(255);
  stroke(32);
  noFill();
  randomSeed(37196299);

  float xPos   = 200;
  float yPos   = 200;
  float sSize  = 200;

  vQuads(xPos, yPos, sSize, 5, 6, 0.5); // <-- half of the calls to vQuad will draw a quad in this case
}

7. Finishing Off - Space and More Randomness

What we have, looks quite a lot like Des Ordres, but still a little too ordered and a bit squished up.

                void vQuads (float x, float y, float s, float r, int n, float p) {

                  s=s*0.9;  // <-- Make the vQuads a little smaller - this could of course be set by a parameter for scaling
                  
                  for (int i=1; i<=n; i++) {
                    if (random(0, 1) <p) {
                      vQuad(x, y, i*s/n + random(-5,5), r); // <-- add randomness to spacing - again paramterisable!
                    }
                  }
                }
