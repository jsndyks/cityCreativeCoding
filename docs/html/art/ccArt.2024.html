This session shows you how you can apply some of your recently developed programming skills to be more artistically expressive. It demonstrates how randomness can be used to generate patterns and how sound, images and video can be incorporated into your sketches to add richness to them.

By the end of this session you should be able to:

  • Use randomness in your sketches to add visual interest and variety;
  • choose whether or not you wish to create an 'artwork' for your BootCamp demo;
  • import libraries into Processing;
  • be able to play samples and songs in your sketches;

1. Art, Artists and Computer Science

As students who have come to City, University of London to study for a degree in the Department of Computer Science, you may be wondering what has art got to do with your studies. Perhaps you have arrived here with qualifications in the sciences, mathematics or technology. As you learn and enhance your coding skills, most of your effort is likely to be in developing a structured, logical way of thinking. You will learn new technical skills, new programming constructions and approaches. You will learn to apply those skills in a systematic and rigorous way.

Yet there is also a space for creative thinking and expression. Sometimes this might have very direct application such as in the development of a game or ideas for a new apps. But creative thinking can also help in problem solving, in developing your own 'voice' or approach to computer science. In this session we will look at some of the ways people have mixed art with computer science to express themselves creatively as well as expand the range of tools available to you in Processing to express yourself.

For an enlightening account of this blend of 'computer science thinking' and expressive visual art, have a read of this interview with Jared Tarbell who produced many artworks with Processing, including Substrate shown above.

Processing was originally created for the purpose of providing artists with access to the computational space. If you watched the Hello World! Processing documentary, you will have heard Casey Reas, one of the original designers of Processing, explain that the idea was to allow new programmers to create a 'sketch book' of quickly generated programs that could explore different ideas. This digital sketchbook provides a medium for the programmer similar to a painter's oil and canvas.

You can see some examples of how programmers and artists have mixed the rigour of programming with the creativity of design, artistic expression and experimentation with Processing:

  • Gallery of Computation — Jared Tarbell's beautiful sketches created with Processing.

  • Bees and Bombs — Processing examples with an emphasis on geometric animations and optical illusions.

Copyright, plagiarism and inspiration

Browsing the internet for examples of what others have done with Processing can be an effective way of gaining inspiration for your own work and getting a sense of programming possibilities. However you must be aware of two important rules when making use of the works of others:

Firstly, do not infringe copyright. If you wish to use, for example, a sample from someone's musical creation, are there copyright restrictions placed on the work that would prevent you from doing this? It is important to respect copyright because that will allow you to protect your own work should you create something original.

Under the Copyright, Designs and Patents Act, 1988, Section 32(2) of the Act allows copying of a sound recording, film or broadcast provided the copying is:

  • (i) done by a person giving or receiving instruction, and
  • (ii) accompanied by a sufficient acknowledgement

So this means that in practice, you can use copyrighted material for learning purposes as part of your education, but should not share copyrighted materials more widely.

A safer approach to using media created by others is to use materials that have licenses that permit or encourage sharing. In particular, media distributed under the creative commons (cc) is a good source that shouldn't have copyright problems. A useful site for searching for creative commons material is search.creativecommons.org.

Secondly, do not plagiarise others' work. Plagiarism means copying someone else's work, such as their code, or design without making it clear that it is their work, not your own. In other words, if you use someone else's work as a basis for your own, you should make it clear, through attribution, where that source material came from and make it easy for the reader to know what is your own work and what belongs to others. Apart from being wrong, copying others' material without understanding exactly how it works is a very poor way of learning and is unlikely to make you a better programmer.

2. Randomness

Adding some randomness to your sketches is one way of adding visual interest. We've seen several examples of this already, such as when we generated moving balls that had a random velocity. The Processing method random() is a handy way of generating the random numbers we might use in our sketches. It normally takes two parameters specifying the range of numbers from which a random value is generated. For example: 

float xPosition = random(7,101);
would generate a random number somewhere between 7 and 101 and store that number inside the variable xPosition.

To see how just a little randomness can add interest to a sketch, let's consider the following simple sketch that draws 8 squares in a row. This is very similar to the squares example we have seen before, but in this case we will use the quad() rather than the rect() method. quad() takes 4 pairs of parameters, each identifying the position of a corner of the shape. We use a simple for loop to draw 8 of these squares, each with a width and height of 90 pixels (stored inside the variable sSize): 

void setup()
{
  size(860, 200);
}

void draw()
{
  background(255);
  noStroke();
  fill(0,0,200,100); // Transparent blue

  float xPos   = 40;
  float yPos   = 50;
  float sSize  = 90;

  for (int i=0; i<8; i=i+1)
  {
    quad(xPos,         yPos,
         xPos + sSize, yPos,
         xPos + sSize, yPos + sSize,
         xPos,         yPos + sSize);

    xPos = xPos + 100;
  }
}

quad() corner positions

The reason we have used quad() rather than rect() in this example is that it allows us to control the position of each corner of each square independently. This is where we can add a simple random number to the position of each: 

for (int i=0; i<8; i=i+1)
{
  quad(xPos + random(-10,10),         yPos + random(-10,10),
       xPos + sSize + random(-10,10), yPos + random(-10,10),
       xPos + sSize + random(-10,10), yPos + sSize + random(-10,10),
       xPos + random(-10,10),         yPos + sSize + random(-10,10));

  xPos = xPos + 100;
}

The only changes made to the sketch are to add random(-10,10) to each x and y position for each corner. If you were to run the sketch though, the result might be a little unexpected (click the sketch to the right).

The squares appear to wobble about rather than produce the static irregular shapes we might have expected. This is because as the draw() loop repeats 60 times every second, new random numbers are generated which are then used to reposition each square's corner points.

This may be a desirable outcome depending on what you are trying to achieve, but if you want your shapes to stay still, you have a number of choices. The first might be to stop the sketch from repeating the draw() code. We can turn off the 60 times a second repetition with a single simple command noLoop(). This is a nice simple option, but has the disadvantage that if we want other parts of our sketch to move, turning off the looping in draw() will prevent this.

An alternative is that we can force Processing to generate the same set of random numbers each time the draw() loop is called. This can be achieved using the randomSeed() method. For example, including the line 

randomSeed(1234);
at the start of draw() will generate the same sequence of random numbers every time draw() is called. You can change the number 1234 to whatever you like and each seed value will generate its own unique sequence of random numbers. This is a useful way of providing the same sequence of random numbers every time you run your sketch.

To see some examples of how you might extend the idea of randomly shifting the corners or appearance of shapes you may wish to look at the Tile Generator project suggestions. Other forms of Randomness

The random() function is very simple to use and works in many different situations. One of its drawbacks for using this method for drawing is that every time a new random number is generated, the new number is completely independent of the one before it. This means that if, say, position of a shape is set using random(), it can appear to jump around the screen in a very distracting way: 

void setup()
{
  size(400,400);
  fill(0,0,100);
  noStroke();
}

void draw()
{
  background(255);
  float xPos = random(0,width);
  float yPos = random(0,height);
  circle(xPos,yPos,50);
}

Randomly positioned circles using random() (click to activate)

An alternative is to use Processing's noise() method instead of random(). This also generates random numbers, but in this case, each subsequent random number can be made to be a little similar to the previous one in the sequence. To see how this works, see the noise() reference page.

Although noise() can produce random numbers (between 0 and 1), it takes one or more parameters and will produce the same random number for any given parameter value. But if we provide something like the frameCount as a parameter, we can make that random number change slowly on each redraw.

Below is a simple modification to the randomly positioned circle sketch that uses noise() instead of random() to generate a much more smooth random movement. 

void setup()
{
  size(400,400);
  fill(0,0,100);
  noStroke();
}

void draw()
{
  background(255);

  float smoothness = 0.05;   // Try changing this number between 0 and 1.
  float xPos = noise(frameCount*smoothness,0)*width;
  float yPos = noise(frameCount*smoothness,1)*height;
  circle(xPos,yPos,50);
}

3. Working with Sound

For the first week we have been working exclusively with graphics in our Processing sketches. It is possible to use other media too, and in particular sound and video. In order to extend Processing to do this, we need to make use of the ability to import libraries into our sketches. A library is simply an extra set of classes written by someone else that can be bundled in with your own sketch. There are many libraries available in Processing — see the Processing library documentation.

3.1 Sound Basics

To work with sound in your sketches, one of the easiest libraries to work with is the Processing Sound library. To install the library, select the Sketch->Import Library -> Add Library... menu item and select Sound authored by The Processing Foundation. You only need do this once to store the library on your computer. It contains functionality to play sound files (e.g. samples or MP3 music files), generate synthesised sounds and monitor sound input.

To see how to use the sound library, let's consider a simple sketch that plays some sound samples whenever certain keys are pressed: 

import processing.sound.*;   // Import the library that does the sound handling.

SoundFile kick, snare, hiHat;  // Each sample in its own object.

void setup()
{
  size(300, 200);

  kick  = new SoundFile(this,"kickDrum.wav");
  snare = new SoundFile(this,"snareDrum.wav");
  hiHat = new SoundFile(this,"hiHat.wav");
}

void draw()
{
  noLoop();		// Stops Processing from redrawing at 60 fps.
}

void keyPressed()
{
  if (key == ' ')
  {
    kick.play();
  }
  else if (key == 's')
  {
    snare.play();
  }
  else if (key == 'h')
  {
    hiHat.play();
  }
}

Keyboard activated drum kit (click to activate)

In terms of programming, there is very little that is new here. What you may not have seen previously is the first line import processing.sound.*;. The import statement is used whenever you need to import a library that is not part of the core Processing language. In this case, the full name of the library is processing.sound and the * ('wildcard') at the end simply indicates that we wish to be able to use any of the classes contained within the library.

The three lines in setup() each initialise new objects kick, snare and hiHat, all of which are of type SoundFile. Just like we have previously declared variables to store numbers or colours or text, this type is a variable capable of storing a complete sound sample. The keyword this tells the class that is it 'this' sketch that needs to load the samples, and the second parameter identifies the names of the sound files that store the samples.

If you are going to work with sound files, you need to place them in to your sketch somehow. The easiest way is to drag the file with the mouse into the Processing code window. This will create a local copy of the dragged file that sits inside a data folder in the sketch location.

To run the drum kit sketch above, you will need to drag three sample files, one for each drum/cymbal type into the sketch. You can download those three sample files, all of which have creative commons licenses, by right-clicking them here:

  • kickDrum wav file
  • snareDrum wav file
  • hiHat wav file

If you have difficulty downloading these files, get them in a ZIP archive from here : Sound Files.

The remains of the drum kit sketch just uses Processing's keyPressed() method to respond to presses of the spacebar, 'S' and 'H' keys to trigger the bass drum, snare drum and hi-hat cymbal sounds respectively.


While we can use interaction to trigger the drum samples, you might want to code when samples are played within your sketch itself. Or perhaps mix interaction and pre-coded sample triggers. In the example to the right we use Processing to create a simple drum machine.

This approach provides an interesting one on which to base a demo.

4.1 Longer music tracks

The approach above, where a sample is played in its entirely, when triggered works well for short samples. For longer samples, such as entire songs, we may wish to play only part of the sample, or pause it midway. The SoundFile class offers some useful methods for doing this.

The example below shows a simple sound player that allows a song to be played and paused. It does this by keeping track of whether the song is being played or not with a Boolean variable isPlaying and calls the methods from the sound library, play() and pause() to turn the music on and off. 

import processing.sound.*;   // Import the library that does the sound handling.

SoundFile song;     // The extended song to play
boolean isPlaying;

void setup()
{
  size(400, 250);
  textSize(36);
  textAlign(CENTER,CENTER);
  song  = new SoundFile(this, "loop7.mp3");
  song.play();
  isPlaying = true;
}

void draw()
{
  background(0);
  String message;
  if (isPlaying)
  {
    message = "Playing";
  }
  else
  {
    message = "Paused";
  }
  text(message,width/2,height/2);
}

void keyPressed()
{
  if (key == ' ')  // Toggle pause on or off.
  {
    if (isPlaying)
    {
      song.pause();
      isPlaying = false;
    }
    else
    {
      song.play();
      isPlaying = true;
    }
  }
}

To load the file loop7.mp3, it must have previously been dragged into the code window so a copy of it is accessible to the sketch (if you want to run the sketch yourself, you might want to find your own .mp3, .wav or .aiff file and drag it into the sketch).

4.2 Responding to Sound

It would be possible to trigger samples or play/pause music files in response to something other than key presses, such as mouse movements or colliding shapes. But as a final example of working with sound, we will draw some simple lines in response to the volume of the sound being played.

To do this we will get the sound library to perform a Fast Fourier Transform (FFT) on the sound file. This is a way of extracting the amplitude (volume) of the sound at different frequencies, allowing us, for example to measure how much bass there is in the sound compared to higher frequencies such as cymbal or hi-hat sounds.

To do this, we use the sound library's FFT class to do all the hard work for us. We use the array of amplitudes it generates (called spectrum) to create a shape representing the frequencies of the sound in real time. 

import processing.sound.*;

FFT fft;                                 // Fourier transform of the sound signal
int NUM_BANDS = 256;
float baseLine;                          // Position of the baseline.

void setup()
{
  size(532, 250);
  background(4, 60, 5);
  baseLine = 20;

  // Load the sound file.
  SoundFile sound = new SoundFile(this, "loop7.mp3");
  sound.play();

  // Tell the Fast Fourier Transformer to process the song file.
  fft = new FFT(this, NUM_BANDS);
  fft.input(sound);
}

void draw()
{
  // Draw a dark green slightly transparent background
  noStroke();
  fill(4, 60, 5, 5);
  rect(0, 0, width, height);

  // Draw bright green lines to represent volume
  stroke(142, 187, 82);
  strokeWeight(2);

  fft.analyze();    // Analyse the current sound being played.

  beginShape();
  for (int i=0; i<fft.spectrum.length*0.4; i=i+1)
  {
    float xPos = map(i, 0, fft.spectrum.length*0.4, 10, width-10);
    float yPos = map(sqrt(fft.spectrum[i]), 0, 1, baseLine, baseLine-height*0.7);
    vertex(xPos, yPos);
  }
  endShape();

  // Move the baseline down the screen every frame until it gets to bottom.
  baseLine++;
  if (baseLine > height-10)
  {
    baseLine = 20;
  }
}

The for loop iterates though the first 40% of the current spectrum produced by the FFT. As the value of i increases in the loop, so the value of fft.spectrum[i] represents the amplitude of sound at increasingly higher frequencies. These amplitude values that are between 0-1 are then scaled to be between 0 and 0.7 times the height of the sketch using processing's map() method.

The example above takes the sound input from a SoundFile (stored in a sound object). We can change the input to be the computer's internal microphone so that we visualize any sound it picks up.
To do this, simply replace the SoundFile with AudioIn and call its start() method instead of play(): 

//SoundFile sound = new SoundFile(this, "loop7.mp3");
//sound.play();

// Link to the computer's mic
AudioIn sound;
sound = new AudioIn(this, 0);
sound.start();

There is plenty more to discover with the Processing sound library should you wish to do more with sound. If you are interested in doing this for your demo, have a browse of the sound library documentation which includes some examples.

5. Working with Video

Challenge - Working with Video
This section introduces capability for using and manipulating video in Processing.

Once again, there are some amazing opportunities here, but we do not consider this to be core activity.

Feel free to head straight to Section 6 unless you want to learn more about video and are perhaps thinking about using it in your demo.

Finally, we will briefly consider how to display video files in Processing. Again we will make use of a library that does most of the hard work for us. As before, the very first time you wish to use a new library, you need to download it and install from the Sketch -> Import Library... -> Add Library... menu, this time selecting Video by the Processing Foundation. In your code, the library needs to be imported with the statement import processing.video.*;, which tells Processing that we wish to make use of any of the classes stored in the processing.video package.

As with sound files, you will also need to drag a video file into your sketch so that Processing knows where to look when reading the file.

The steps for actually displaying the video are quite simple, and very much like those taken to play a sound file. It is best illustrated with an example, in this case a simple sketch that will read in a video file (downloaded from HelloWorld! Processing) and play it in the sketch: 

import processing.video.*;    // Needed to display video.

Movie myVideo;                // Stores the video object.

void setup()
{
  size(640, 360);
  myVideo = new Movie(this, "HelloWorldClip.mov");
  myVideo.play();
}

void draw()
{
  // Display the video at position (0,0) at its natural size.
  image(myVideo, 0, 0);
}

// This is needed to read in the movie while it is playing.
void movieEvent(Movie videoBuffer)
{
  videoBuffer.read();
}

After importing the video library, the first line creates a Movie object which we've called myVideo in this example. Remember Movie (capital 'M') is the name of the class that can store the video data and contains methods that operate on the data and myVideo is the object that is an instance of the Movie class.

myVideo is initialised inside setup() where it is read from the file HelloWorldClip.mov. This needs to correspond to the name of the file you have dragged into the Processing window. If you use a different video file, you would need to edit the code to use the relevant name. Once loaded, the video is started by calling the .play() method of the object.

The display of the video is done inside the draw() method using the image() command. This can take either 3 or 5 parameters. In this case, we use the 3-parameter version that firstly names the Movie object to display (myVideo in this example), then the x and y position of the top left of the display position, just as we might if displaying a rectangle.

The only other method we need to add to our sketch when dealing with videos is one called void movieEvent(). This is a special method that Processing will look for whenever it needs to read in a bit more of the video from the file.

Controlling the size and position of a video display

We can build some quite interesting display designs with videos because once a video is read, it can be displayed as many times as you wish and at any size. For example, the sketch below is exactly the same as our simple video sketch above except for the highlighted lines that this time use two nested loops to display 25 copies of the same video: 

import processing.video.*;    // Needed to display video.

Movie myVideo;                // Stores the video object.

void setup()
{
  size(800, 500);
  myVideo = new Movie(this, "HelloWorldClip.mov");
  myVideo.play();
}

void draw()
{
  background(0);

  for (int row=0; row<5; row=row+1)
  {
    for (int col=0; col<5; col=col+1)
    {
      image(myVideo, 10 + col*160, 10 + row*100,140, 90);
    }
  }
}

// This is needed to read in the movie while it is playing.
void movieEvent(Movie videoBuffer)
{
  videoBuffer.read();
}

We can combine video playback with any number of other Processing graphical display options. For example it would be possible to display the video at the current mouse position, or to change its size according to the mouse position.

The final example below shows how the position of the video can move randomly using the noise() command, just as we did with the moving circle at the start of this session. Additionally, we use the tint() command to make the video display transparent (you will learn more about this in the session on image processing). 

import processing.video.*;    // Needed to display video.

Movie myVideo;                // Stores the video object.

void setup()
{
  size(1200, 800);
  background(0);
  myVideo = new Movie(this, "HelloWorldClip.mov");
  myVideo.play();
  }

void draw()
{
  float px = map(noise(frameCount*0.01, 1), 0, 1, 0, width-myVideo.width);
  float py = map(noise(frameCount*0.01, 2), 0, 1, 0, height-myVideo.height);
  tint(255, 4);      // Make the video very transparent.
  image(myVideo, px, py, 600, 380);
}

// This is needed to read in the movie while it is playing.
void movieEvent(Movie videoBuffer)
{
  videoBuffer.read();
}
