Copyright

As responsible Computer Scientists, it is essetial that you use resources that you find online in accordance with the conditions of use under which they are made available.
You need to operate within Copyright Law, just as you need to operate within the other laws that we use to make society run smoothly!
We would recommend being as explicit about this as possible - it's simply good practice.

We encourage you to use materials that have been created by others and made available under licences that permit or encourage sharing.
Media, including images and audio distributed under the creative commons (cc) are a good place to start.
A useful place to searching for creative commons material is the Openverse.
According to the site ...

"All Openverse content is under a Creative Commons license or is in the public domain."

Creative Commons licenses have various flavours - and we strongly recommend that you follow ðŸ‘ˆ the link and have a look at them.
Examples include ...

  • CC BY - which allows you to use, modify and resdistribute content, but requires you to give credit to the author and describe any changes that you have made to the material.
  • CC BY ND - which allows you to use and resdistribute content, and once again requires you to give credit to the author, but does not allow you to make modifications to the material.
  • CC BY ND - which allows you to use, modify and resdistribute content, requires you to give credit to the author, but does not allow you or others who use your work for commercial gain.

You need to know under which license you are operating and ensure that you comply with its terms.

Sources

Many sites that provide media and allow you to search for it give you facilities for specifying usage rights in the search. Most also make it clear how accreditation is to be given. Now, it turns out that if you Go Premium you can use some of the Freepik images without attribution. As you are studying, I see no advantage to this. When discussing the pros and cons with students in class, somebody asked me:

But how will you [lecturers] know whether we have gone premium or not?

Well, we may not. Or we may check, and ask for evidence. But in a way, it's the wrong question. We need to know the sources of all the materials and media that you use. If you don't mention a source, then our assumption is that you have made the thing : code, essay, media, whatever. So you always need to be clear about where your materials come from. If you had paid for a premium account, I would suggest using the image as follows:

When you drive a car in a residential area, somebody might not see you doing more than 30mph, but are not being fair others who you put at risk.
When you fancy a muffin from the coffee shop, somebody might not see you pinch one without paying if you are quick, but you are not being fair to those from whom you take.
But these are not behaviours that operate within the law - whether anybody sees you or not. It's not about whether you get caught, it's about the way you behave and the harm you may do to others. Using the work of others is fine, as long as you do so within the restrictions that apply. So we urge, and indeed require you to operate within information laws, and this means that you must understand and respect copyright, acknowledge sources, and use content that you have acquired from elsewhere responsibly. And you must be explicit about this.

In short:

  • Be responsible - understand the law and operate within it
  • Be explicit - let whoever sees your work know exactly where it is originates from
  • Be cautious - if you are not sure whether you can use the material, don't use it;
    if you are not sure whether your acknowledgment is sufficiently explicit, be more explicit

            // -- Sprite class for showing animated images and moving them in 1 dimension (x):

            class Sprite {
            
              int        imageNum;
              PVector    pos, siz;
              PImage[]   images;
              int        animImageNum;
              int        changeSpeed;
              float      vel = random(-10,10);
              int        iWidth, iHeight, xOffset;
              boolean    backwards = true;
              boolean    flipDirection = true;
              String     license, source;
            
              Sprite(int i, PVector p, PVector s) {
            
                // -- key variables that define look and behaviouir of sprite:
                imageNum    = i;
                images      = animImages[i];
                iWidth      = images[0].width;
                iHeight     = images[0].height;
                pos         = p;
                siz         = s;
                changeSpeed = int(random(100,500));
                xOffset     = int(s.x/2);
                
                // -- whether or not we flip the image, and change this on bounce is randomised to give some variation:
                if (random(-1,1) >0) backwards     = false;
                if (random(-1,1) >0) flipDirection = false;
                
                // -- add image rights from imageRights array:
                license = split(imageRights[i],",")[0];    // <- split the imageRights string and use the first element
                source  = split(imageRights[i],",")[1];    // <- split the imageRights string and use the second element
              }
            
              // -- move the sprite in x according to velocity in x:
              void move () {
                pos.x += vel;
              }
            
              // -- update the image to use in the sprite based upon time sketch has been running (see 'modulo' ref doc): 
              void animate () {
                animImageNum = (millis()/changeSpeed)%images.length;
              }
            
              // -- use translation to position and possibly flip image:
              void draw() {
                pushMatrix();
                translate(pos.x,pos.y);
                if (backwards) scale(-1,1);
                image(images[animImageNum],-xOffset,0, siz.x,siz.y);
                popMatrix();
              }
            
              // -- show image rights if mouse is inside image:
              void checkMouse() {
                if (mouseX > pos.x - xOffset && mouseX < pos.x + xOffset) {
                  if (mouseY > pos.y && mouseY < pos.y + siz.y) {
                    textAlign(CENTER);
                    fill(32,128);
                    textSize(18);
                    text(license,width/2,height-48);
                    textSize(10);
                    text(source, width/2,height-32);
                  }
                }
              }
            
              // -- invert velocity in x if image hits the side of the canvas, with buffer of 'd':
              void checkBounce(int d) {
                if (pos.x - xOffset < d) {
                  pos.x = d + xOffset;
                  vel = vel*-1;
                  if (flipDirection) backwards = !backwards;
                }
                if (pos.x + xOffset > width-d) {
                  pos.x = width - d - xOffset;
                  vel = vel*-1;
                  if (flipDirection) backwards = !backwards;
                }
              }
            }

  // -- Script to create animated figures from 4*2 grids of images showing animated sequences
  // --  such as those avaialble at Frepik - https://www.freepik.com/serie/31630110
  // -- Jason DYKES : 13 October 2022
  
  String []   imageFiles   = {
    "roll7842533.jpg", // -- an array with the names of the composite image files I have downloaded:
    "jump7849353.jpg",
    "ball7842745.jpg",
    "long7836915.jpg",
    "pack7837076.jpg"
  };
  String []   imageRights  = {
    // -- an array with details of the image rights and sources:
    "Image by Freepik,https://www.freepik.com/free-vector/flat-design-animation-frames-element-collection_31630110.htm",
    "Image by Freepik,https://www.freepik.com/free-vector/flat-design-animation-frames-element-collection_31630040.htm",
    "Image by Freepik,https://www.freepik.com/free-vector/flat-design-animation-frames-element-collection_31630041.htm",
    "Image by Freepik,https://www.freepik.com/free-vector/flat-design-animation-frames-element-collection_31630108.htm",
    "Image by Freepik,https://www.freepik.com/free-vector/flat-design-animation-frames-element-collection_31630042.htm"
  };
  
  PImage []   sourceImages;             // -- an array to contain the source images (the composites)
  PImage [][] animImages;               // -- an array of arrays containing the split images (the individuals)
  
  // -- Animated Sprites (see Sprite class) will be added to this ArrayList on mouse click.
  ArrayList sprites = new ArrayList();
  
  int     iRows  =   2;                  // -- how many images are there in the composite : rows
  int     iCols  =   4;                  // -- how many images are there in the composite : cols
  int     iWide  = 200;                  // -- how wide are the images?
  int     iHigh  = 275;                  // -- how high are the images?
  int     sWide  =  90;                  // -- how wide do we want the Sprites to be?
  float sRatio =  iHigh/float(iWide); // -- what aspect ratio are the Sprites?
  int     sHigh  =  int(sWide*sRatio);   // -- how high are the Sprites?
  
  boolean loop = true;                  // -- run the draw loop or not?
  boolean showImageMode = false;        // -- show images or animate?
  
  
  void setup () {
  
    size(500, 800);
  
    // -- load the images into an array - this just reads the composites from file, and splits the images up :
    loadAnimImages();
  
    // -- make a couple of sprites :
    for (int i=0; i<2; i++) {
      int r = floor(random(0, animImages.length));      // -- choose random image
      // -- create Sprite at (constrained) random y position add it to ArrayList
      addSprite(r, width/2, int(random(sWide/2, height-sRatio*sWide)), sWide, sHigh);
    }
  
  }
  
  
  void draw() {
    if (showImageMode) {
      background(250);
      drawImages(0.75);
    } else {
      background(232, 242, 244);
      for (Sprite s : sprites) {
        s.move();
        s.checkBounce(5);
        s.animate();
        s.draw();
        s.checkMouse();
      }
    }
  
    if (!loop) {
      noLoop();
    } else {
      loop();
    }
  }
  
  // -- click the mouse to add a sprite :
  void mouseClicked() {
    int r = floor(random(0, animImages.length));
    addSprite(r, mouseX, mouseY-sHigh, sWide, sHigh);
  }
  
  // -- press a key to stop the animation :
  void keyPressed() {
    if (key == 'i') {
      showImageMode = !showImageMode;
    } else {
      loop = !loop;
      loop();
    }
  }
  
  // -- method thay generates a sprite:
  void addSprite (int i, int x, int y, int wi, int hi) {
    // -- creates a new sprite and adds it to the sprites ArrayList:
    sprites.add(new Sprite(i, new PVector(x, y), new PVector (wi, hi)));
  }
  
  // -- process to load and cut images - relies on globals!
  void loadAnimImages() {
    sourceImages  = new PImage [imageFiles.length];
    animImages    = new PImage [imageFiles.length][iRows*iCols];
  
    // -- parameters to work out where to cut individual images out of composite with PImage.copy:
    int iBorderX = 33;  // --  width of border around composite image
    int iBorderY = 12;  // -- height of border around composite image
    int iGapX    = 10;  // -- gap in x between composite images
    int iGapY    =  0;  // -- gap in y between composite images
  
    int i=0;
    for (String f : imageFiles) {
      sourceImages[i] = loadImage(f);
      println("\t->\timage loaded:", f);
      //int iStepX = int(sourceImages[i].width  / float(iCols));
      //int iStepY = int(sourceImages[i].height / float(iRows));
  
  
      int a=0;                                // -- animated image number (from composite)
      for (int r=0; r<iRows; r++) {
        int y =iBorderY + r*(iGapY+iHigh);    // <- y positon of part of composite image to copy (top left)
        for (int c=0; c<iCols; c++) {
          int x =iBorderX + c*(iGapX+iWide);  // <- x positon of part of composite image to copy (top left)
  
          // -- make a new mini image to fill with the copy from the composite ...
          animImages[i][a] = createImage(iWide, iHigh, RGB); // < -- works
  
          // -- copy the section from the source image ...
          animImages[i][a].copy(sourceImages[i], x, y, iWide, iHigh, 0, 0, iWide, iHigh);
  
          a++;  // <- counts mini animated image files (cuts from the composite)
        }
      }
      i++;      // <- counts source image files
    }
  }
  
  // -- checks that image cuts are reasonable - press 'i'
  void drawImages(float scaler) {
    noFill();
    strokeWeight(0.5);
    stroke(128,128,255);
    pushMatrix();
    scale(scaler);
    for (int i=0; i<sourceImages.length; i++) {
      float x = sWide*(i+0.05)*1.05;
      for (int s=0; s<animImages[i].length; s++) {
        float y = sHigh*(s+0.05)*1.05;
        //println("i:", i, "s:", s);
        image(animImages[i][s], x, y, sWide, sHigh);
        rect(x, y, sWide, sHigh);
      }
    }
    popMatrix();
  }
