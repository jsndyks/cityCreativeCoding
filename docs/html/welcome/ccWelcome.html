Nice NoteBooks

At the start of #CreativeCoding you will be given a nice quality notebook with graph paper pages and a couple of pens. These are to help you in #CreativeCoding and beyond.

We want you to sketch ideas and take notes as you learn as this helps you think things through, plan your learning and work things out. You are going to need to be good at all of these things to succeed in your degree.

You also need to collaborate and communicate - get to know people and work with them. So share your coloured pens with others in the class if you need more colours. Hunt around - find somebody who has the green that you need. See if you can make a üåà through some cross-class collaboration - you'll get to meet a few of your colleagues.

Your notes can be digital or physical - but make sure that you know where they are and how to access them when the teachers are around to discuss your work with you.

Academic Autonomy

When #CreativeCoding appears on the timetable, we will usually have two hours together in class to introduce and discuss key topics.

Make sure you are seated before the time that the lecture is scheduled to start. Be sure to have your laptops open and ready as we will be trying out coding tasks during the lectures. Find the Moodle page for the lecture before things begin as you may need to follow links, copy code and download data.

After (or sometimes before) the lectures, you will have a Practical Class scheduled, led by one of our Teaching Assistants (TAs).

But, you will not complete all of the #CreativeCoding work in as single hour - there is more to do than this.

We expect you to spend 3 or 4 hours outside of the lectures completing the exercises, following links, checking documentation, discussing progress, etc. And we recommend that you do so predominantly on the day of the lecture or as soon afterwards as you are able.
We actually encourage you to work through the notes and try the tasks before the lecture if you are able to do so.

Your practical classes are not always scheduled immediately after the lecture, so we strongly recommend that you work on the #CreativeCoding activity before, after and around the practical class. This means that you can come to class prepared to discuss your progress, efforts, plans and attempts to solve the tasks with our teaching staff.

Put time in your diary around the class to make the most of the materials, and the classes.

Yeah, that's right ... we want you to ü¶ú.

Now, learning Parrot Fashion involves learning ...

In the manner of a parrot;
      with thoughtless repetition of what others have said
Wiktionary (2024)

But we are asking you to put effort into doing the very opposite of that.
We would like you to ... plan, attempt, run, observe, think and then perhaps talk, before doing the same thing again in a progressive learning cycle: PAROT Talk

In completing a small #CreativeCoding coding task, you might ...

  • Plan - your approach, with some sketches and thinking about what you are trying to achieve, and what your approach may be
  • Attempt - to solve the problem based upon this planning, for example by writing some carefully considered and fully documented code
  • Run - the approach, try it out, for example in Processing by clicking the 'run' button
  • Observe - what happened? Was it what you expected, or something else?
    Watch carefully, check any debug messages, and any reporting that you have added to the approach (if you didn't add any at first, add these to your plan next time).
  • Think - can you explain what happened, why things went wrong (if they did)?
    How does reflecting deeply on your attempt and subsequent observation inform your planning for next time?
    What have you learned?
    What do you not understand?
  • Talk - communicate with colleagues by explaining your thinking, showing them your attempt and gaining alternative and perhaps even diverse perspectives.
    Talk to your teachers, electronically or in person. Check any documentation.

... and then start another cycle until the ask is complete.

We would like you to try this when you undertake #CreativeCoding tasks.
For example: copy this code, into a Processing sketch and run it.
Change the five global variables that control the way in which the sketch changes over time.
Every time you make a change, go through a PAROT (talk) cycle, and if things did not turn out as you expected, go again into cycle two.
Several cycles should help you understand what each of the global variables do, and how the Processing rect() and frameRate() methods work and how the mouseX, mouseY and frameCount variables vary.

// -- global variables : Change these FIVE in turn, and in combination.
// -- Use PAROT talk to learn their effects.
                    
float mouseXDiv = 1;           // <-- change number that mouseY is divided by
float mouseYDiv = 1;           // <-- change number that mouseX is divided by
float rectRatio  = 0.5;        // <-- change the aspect ratio of the rectangle
float widthFactor = 2;         // <-- change the speed at which the rectangle changes size
float framesPerSecond = 60;    // <-- change frameRate from 1-60
                    
                    
void setup() {
  size(400, 300);
  background(255, 236, 149);  // <-- change each of the colour components between 0 and 255
  fill(203, 118, 122);        // <-- change each of the colour components between 0 and 255
}
                    
                    
void draw() {
  frameRate(framesPerSecond);  // -- sets number of frames per second.
                    
  // -- calculate position and dimensions of rectangle depending on global variables:
  float x = mouseX / mouseXDiv;
  float y = mouseY / mouseYDiv;
  float rectWidth = frameCount / widthFactor;
  float rectHeight = rectWidth * rectRatio;
                    
  // -- draw a rectangle at x,y with the specified height and width:
  rect(x, y, rectWidth, rectHeight);
}

#BootCamp

#CreativeCoding is based on the materials and approaches used in our #BootCamp activity. For many years we have run a two-week intensive introduction to coding at the start of year 1. But we have changed things a little this time as we redesign our learning activities. #BootCamp as intensive, perhaps too intensive, and we didn't like the lack of time, the negativity around the term, and the lack of clearly defined outputs.

Nevertheless, our #BootCamp experiences, and those of former #BootCamp students are worth considering as we begin to learn through #CreativeCoding.

We asked some former students who survived #BootCamp what their advice would be for anybody about to start the course.
Here is their guidance and feedback for you:

Start to learn the main concepts of conditions early on in the course:
if, while, do ... while, for loops.
This helps immensely in most of the coursework related to Java

... and get a hand from teaching staff if you find these concepts challenging.

Try to grasp the notion of classes and objects, but don't worry if you can not, as this is taught several times during the first year.

... once again, discuss these challenging but important concepts with staff as we work through week 1.

If you already have experience of coding then some of the concepts and exercises will seem very simple but its always good to get practice in and try to challenge yourself on some of the tasks. There are always ideas for taking things further - maybe add a new feature or additional aspects to the sketch you are working on.

... if you know how to code, just have fun coding new stuff and seeing what Processing and its libraries can do - look for the red 'challenges' to guide you.

To students who don't have experience with coding don't focus too much on getting the math calculations perfect like the size of a shape to be exactly like in the example and focus on understanding what the code is doing and how it works.

... yes, we do not need you to copy what we have done exactly. Focus on understanding the concepts and not the detail of replication.

Take time to understand how to read the Processing reference documentation as being able to read documentation is a very important skill which I felt wasn't really taught. Get a hand with this too! This is especially the case for those who don't have coding experience it often looks like a whole load of gibberish. Students who don't take the time to look and break it down will not be able to use all of this functionality.

... the Processing reference material is an essential source, so please take this advice and take time to explore and understand it. As ever, teachers online and in class can help you make sense of this.

Inappropriate Intelligence

Of course, computers can do all sorts - they can problem solve, find information and write code for you very effectively.

OpenAI's ChatGPT is very good at writing Processing sketches as you will discover if you try it out or watch the AI at BootCamp video to the right üëâ.

I made this a couple of years ago, and ChatGPT has not become any worse at computer programming since then!

But we'd like you to try to complete the tasks and solve the challenges without using AI please.

Doing so will give you the skills and outlook that you require to solve more complex challenges as you move through your course. We need to know that you have a grounding, a foundation in the kind of creative computing that we are focussing on here.

So let's keep the AI out of #CreativeCoding - we have plenty of use for it, and to learn about it as we go through the degree course, but you will only succeed in the wider degree if you have the skills and experience that we are trying to help you establish in year 1.
And that requires you to take the time to complete these task and exercises, think them through, get them wrong, understand them and painstakingly type the code that is required to do this.

So, once again - keep the AI out of #CreativeCoding if you want to learn the essentials of coding.
We think that you should.
