These exercises develop your ability to use if-statements in your code.

This exercise is an adaptation of the “Heads or Tails” sketch from the lecture: 

float randomNumber;

void setup()
{
  size(200, 200);
  fill(0);
  textSize(48);
  textAlign(CENTER);

  randomNumber = random(2.0);
}

void draw()
{
  background(230, 255, 230);

  String message;
  if (randomNumber < 1.0) {
    message = "TAILS";
  } else {
    message = "HEADS";
  }
  text(message, width/2, height/2);
}
Copy this code into a new sketch. Save the sketch with the name “TwoCoins”.

The aim is to extend this code so that it simulates two coin tosses instead of just one. The possible combinations for two coin tosses are TT, TH, HT, HH. Your sketch should print one of three messages:

  • “TWO TAILS” (combination TT)
  • “TWO HEADS” (combination HH)
  • “MIXED” (combinations TH and HT)
Develop your solution in three stages:

  1. Declare an additional variable at the top of the sketch (call it something like randomNumber2) and initialise it in setup() to a random number in the range 0 to 2. Check that your sketch runs (you won't see any new behaviour yet).

  2. Modify the if-statement in draw() so that the condition tests if both the random numbers are less than 1. You should use the && operator to do this. Modify the message variable assignments so that the first one is “TWO TAILS” and the second one is “WORK IN PROGRESS!”. Check that your sketch runs and behaves as expected. If you run it lots of times you should find that it displays the message “TWO TAILS” roughly a quarter of the time.

  3. Extend the if-statement with extra cases to test for other combinations and set the message variable appropriately. Check that your sketch runs and behaves as expected. If you run it lots of times you should find that it displays “TWO TAILS” roughly a quarter of the time, “TWO HEADS” roughly a quarter of the time, “MIXED” roughly half the time.

In this exercise and the next you will develop an animated ball bouncing inside a box. We start by adapting the scrolling text example from the lecture: 

float xPos; // x-position
float vx; // speed in x-direction

void setup()
{
  size(400,300);
  fill(255,177,8);
  textSize(48);

  xPos = width/2;  // Initialise xPos to centre of sketch
  vx = -2; // Set speed in x-direction to -2 (moving left)
}

void draw()
{
  background(64);

  String message = "The next station is Angel...";

  xPos = xPos + vx;  // Change x-position on each redraw

  text(message,xPos,(height/2) + 20);
}
  1. Copy this code into a new sketch and save it with the name "BouncingBall". Delete the line of code in draw() which creates the String variable message.

  2. Delete the final line of code (text(message, ...) and replace it with code which instead draws a circle centred at position (xPos, height/2). Use a radius of about 50 pixels (you can change this later but, for now, a nice big fat ball makes it easier to see what is happening). Best practice is to add an extra variable radius to your sketch (initialise to 50 in setup()) and use that in draw() instead of the literal 50.

    Check that your sketch works as expected. The ball should start in the centre of the sketch and move slowly to the left, eventually disappearing off screen.

  3. Now we want to make the ball bounce when it hits the edge of the window, rather than go straight through. Add an if-statement at the end of draw() to check if the ball has reached the left-hand edge of the window. Since the ball is moving more than one pixel in each frame, you should test if the edge of the ball is less than or equal to zero (it might actually have started to “penetrate the wall” but don't worry, the visual appearance will be fine).

    If the ball has reached the left-hand edge of the window your code should change the x-speed (variable vx) so that in future frames it will move in the opposite direction. The simplest way to do this is simply to assign a new value to vx which is the negation of its old value: vx = -vx; (so if vx is -2 before this assignment, it will be --2 = 2 afterwards).
    Hint: the left hand edge of a circle is at an x-position which is radius pixels to the left of its centre.
    Check that your sketch works as expected. The ball should start in the centre of the sketch and move slowly to the left. When it reaches the left edge it should “bounce”, reversing direction and moving back across the screen to the right.

  4. Add an extra if-statement to deal with the ball colliding with the right-hand edge of the window. (Or combine this case into the existing if-statement by modifying the boolean expression.)

    Check that your sketch works as expected. The ball should now move backwards and forwards across the window, bouncing when it reaches left and right edges.


In this exercise you add movement in the y-direction to your bouncing ball sketch. You might worry that simultaneous movement in two dimensions will introduce all sorts of new complexities into your code. Luckily for us, we can treat movement in the two directions independently, using very similar code for both cases, and everything works out fine.

  1. Add extra variables yPos and vy to your sketch. In setup(), initialise yPos to be halfway down the screen and initialise vy to 1 pixel per frame.

    In draw(), modify the code which draws the ball so that it uses variable yPos instead of height/2. Next to the assignment which adds vx to xPos, add a new assignment which does the same for vy and yPos.

    Check that your sketch works as expected. The ball should now move in a diagonal direction but it won't yet bounce off the top or bottom edges of the window.

  2. Add code to make the ball bounce off the top and bottom edges. Copy and adapt the code you already have for dealing with the left and right edges, replacing xPos and vx by yPos and vy as appropriate.

    Check that your sketch works as expected. The ball should now move in diagonal directions bouncing of all four walls. Congratulations!

  3. Play around with the initialisation values in your sketch to see how they affect behaviour (window size, fill colour, ball radius, initial position, velocity).

Modify your bouncing ball so that it changes colour each time it bounces. You will need one or more new global (sketch-wide) variables to represent the current colour of the ball (for example you could have three variables, one each for red, green and blue). Then you can modify the code which draws the ball so that it sets the fill colour using these new variables, and modify the code which detects a bounce so that it assigns new values to them.
If you assign random values for red, green and blue, remember that the maximum range for each colour component is 0 to 255.

  1. Write a sketch called “Hunt” which draws a circle at a randomly chosen position. Now modify the sketch so that it only draws the circle when the mouse is inside the circle (so the user has to hunt for the invisible circle using the mouse).

  2. To add a bit of interest to the Hunt game, allow only limited time for the user to find the circle; when time is up, reveal the circle and display a GAME OVER message.
    The variable frameCount keeps track of how many times draw() has been executed since the sketch started running. You can use this to test if the user has run out of time (for example, the default frame rate is 60 frames per second, so five seconds is up when frameCount exceeds 5 x 60 = 300). When the game is over you can freeze the action by setting the frame rate to zero using command frameRate(0).
