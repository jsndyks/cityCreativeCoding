Before you try out these optional exercises, please make sure that you have worked through and understood all of the Decisions : Core concepts and exercises.

These additional optional exercises will help you develop your ability to use if statements in your code.

Two Coins?

This exercise is an adaptation of the “Heads or Tails” sketch from the Decisions : Core activity : 

float randomNumber;

void setup()
{
  size(200, 200);
  fill(0);
  textSize(48);
  textAlign(CENTER);

  randomNumber = random(2.0);
}

void draw()
{
  background(230, 255, 230);

  String message;
  if (randomNumber < 1.0) {
    message = "TAILS";
  } else {
    message = "HEADS";
  }
  text(message, width/2, height/2);
}

Copy this code into a new sketch. Save the sketch with the name “TwoCoins”.

The aim is to extend this code so that it simulates two coin tosses instead of just one.
The possible combinations for two coin tosses are TT, TH, HT, HH. Your sketch should print one of three messages:

  • “TWO TAILS” (combination TT)
  • “TWO HEADS” (combination HH)
  • “MIXED” (combinations TH and HT)

Develop your solution in three stages:

  1. Declare an additional variable at the top of the sketch (call it something like randomNumber2) and initialise it in setup() to a random number in the range 0 to 2.
    Check that your sketch runs (you won't see any new behaviour yet).
  2. Modify the if-statement in draw() so that the condition tests whether both the random numbers are less than 1 with an if statement.
    You should use the && operator to do this.
    Modify the message variable assignments so that the first one is “TWO TAILS” and the second one is “WORK IN PROGRESS!”. Check that your sketch runs and behaves as expected. If you run it repeatedly many times you should find that it displays the message “TWO TAILS” roughly a quarter of the time.
  3. Extend the if-statement with extra cases to test for other combinations and set the message variable appropriately.
    Check that your sketch runs and behaves as expected.
    If you run it lots of times you should find that it displays “TWO TAILS” roughly a quarter of the time, “TWO HEADS” roughly a quarter of the time, “MIXED” roughly half the time.
Bouncing Up & Down

In this exercise you add movement in the y-direction to the bouncing ball sketch that you completed in Decisions : Core.

You might worry that simultaneous movement in two dimensions will introduce all sorts of new complexities into your code. Luckily for us, we can treat movement in the two directions independently, using very similar code for both cases, and everything works out fine.

  1. Add extra variables yPosition and yVelocity to your sketch.
    In setup(), initialise yPosition to be halfway down the screen and initialise yVelocity to 1 pixel per frame.

    In draw(), modify the code that draws the ball so that it uses variable yPosition instead of height/2.
    Next to the assignment that adds xVelocity to xPosition, add a new assignment that does the same for yVelocity and yPosition.

    Check that your sketch works as expected.
    The ball should now move in a diagonal direction but it won't yet bounce off the top or bottom edges of the window.

  2. Add code to make the ball bounce off the top and bottom edges.
    Copy and adapt the code you have already produced for dealing with the left and right edges, replacing xPosition and xVelocity with yPosition and yVelocity as appropriate.

    Check that your sketch works as expected. The ball should now move in diagonal directions bouncing of all four walls. Congratulations!

  3. Play around with the initialisation values in your sketch to see how they affect behaviour (window size, fill colour, ball radius, initial position, velocity).

Aside:

Once you have coded a ball bouncing in a box you will naturally want to add more sophisticated behaviours.
Multiple balls? Balls of different sizes? Balls that collide with and bounce off other balls?
All is possible!

The necessary coding techniques are covered in future sessions.
If you want a sneak preview, have a look at the the Bouncing Balls page in the Processing Resources on Moodle.

Bouncing Colours

Modify your bouncing ball so that it changes colour each time it bounces.
You will need one or more new global (sketch-wide) variables to represent the current colour of the ball (for example you could have three variables, one each for red, green and blue).
Then you can modify the code that draws the ball so that it sets the fill colour using these new variables, and modify the code that detects a bounce so that it assigns new values to them when this occurs.

If you assign random values for red, green and blue, remember that the maximum range for each colour component is 0 to 255.

Modify your colour changing code so that the ball only changes colour on every third bounce.

Modify your code so that the ball speeds up if the cursor / mouse is positioned inside it.

Extension exercises

Hunt!

  1. Write a sketch called “Hunt”, which draws a circle at a randomly chosen position.
    Now modify the sketch so that it only draws the circle when the mouse is inside the circle - so the user has to hunt for the invisible circle using the mouse.

  2. To add a bit of interest to the Hunt game, allow only limited time for the user to find the circle; when time is up, reveal the circle and display a GAME OVER message.

    The variable frameCount keeps track of how many times draw() has been executed since the sketch started running.
    You can use this to test whether the user has run out of time (for example, the default frame rate is 60 frames per second, so five seconds is up when frameCount exceeds 5 x 60 = 300).

    When the game is over you can freeze the action with frameRate() by setting the frame rate to zero using the command frameRate(0).
