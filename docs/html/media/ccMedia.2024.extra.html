These optional exercises give you plenty of practice in, and ideas for, working with sound and video media.

Only attempt them if you have completed the core activity and are excited about trying out some more exciting stuff to push your Processing skills.

These tasks give you some good practice in applying some of the programming skills you have been developing this term.

However, if coding to use sound and video does not particularly interest you, then you do not need to complete these exercises. You can instead work on some of the earlier tasks, or perhaps develop your Dear Data ideas and sketches.


1. More Random, More Interesting?

Take any sketch you have written so far and introduce some new or extra randomness to its design.

Try experimenting with controlling different aspects of the design with random numbers (e.g. size, colour or position of shapes).

Remember you can use random() and or noise() to generate random numbers (see the Randomness section in Art : Core for more detail and a reminder).



2. Noisy Balls

Enhance the Bouncing Ball sketch (reproduced below, but feel free to use your own if you prefer), so that the ball makes a bouncing noise when it changes direction after hitting the edge of the sketch.

Hint: You will need to use the Processing sound library as shown in the lecture notes to load, store and trigger a sample.

For convenience, here are a couple of sound effects you can use, but do use (find, record?) your own if you prefer.

  • boing.wav (right click to download, then drag file into Processing's code window containing your sketch)
  • blop.wav (right click to download, then drag file into Processing's code window containing your sketch)
The main sketch
Ball ball;

void setup()
{
  size(400, 300);
  ball = new Ball();
}

void draw()
{
  background(254, 244, 232);
  ball.draw();
  ball.move();
}
The Ball class
class Ball
{
  PVector position;    // Position of the ball.
  PVector velocity;    // Amount it moves on each redraw.
  float diameter;      // Size of ball.
  color colour;        // Colour of ball.

  // Initialises the ball's state.
  Ball()
  {
    position = new PVector(width/2, height/2);
    velocity = new PVector(random(-2,2),random(-2,2));
    diameter = 30;
    colour   = color(214, 93, 69);
  }

  // Draws the ball at its current position.
  void draw()
  {
    noStroke();
    fill(colour);
    circle(position.x, position.y, diameter);
  }

  // Moves the ball according to its current velocity.
  void move()
  {
    position.x = position.x + velocity.x;
    position.y = position.y + velocity.y;

    if (position.x < diameter/2)
    {
      velocity.x = -1 * velocity.x; // If we are left of sketch move ball right.
    }
    else if (position.x > width-diameter/2)
    {
      velocity.x = -1 * velocity.x; // If we are right of sketch move ball left.
    }

    if (position.y < diameter/2)
    {
      velocity.y = -1 * velocity.y; // If we are at top of sketch move ball down.
    }
    else if (position.y > height-diameter/2)
    {
      velocity.y = -1 * velocity.y; // If we are at bottom of sketch move ball up.
    }
  }
}

3. Audio Visual

Create some form of display that responds to the sound levels of an audio track.

You can use the 'Responding to Sound' example from Sound & Video : Core as the basis of your sketch.

What type of graphics you create in response to the sound signal is up to you.

If your data recording involves sounds you could use these ideas in your Dear Data sketch to make it particularly interesting and perhaps expressive. If the sound visualizer interests you then think about the sounds you could collect, play and visualize in your sketch.



4. Moving Movie

Display a video file so that its position moves with the mouse.

You can use any video file you like, but try not to make it too large otherwise your sketch may load very slowly.

Hint: You can use the simple video sketch in Sound & Video : Core as the basis for this exercise but you will need to use the 5 parameter version of image() to set both the position of the video display and to make the video size smaller than the sketch dimensions.
