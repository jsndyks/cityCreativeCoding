These additional exercises give you practice developing data visualization sketches. The options are almost endless, but considering and attempting some of these challenges may help you develop a Data Visualization project.

Extra Challenge - Live Bike Data Visualization

The bike scheme data that you have used so far is from 2015.

Live bike data is available from from https://www.staff.city.ac.uk/~sbbb717/tfl_bikes/latest-tsv.php. Copy your solution from the previous task and modify it to use this live data.

Hints:

  • Open the latest data file in a browser to see the structure of the data.
  • You can put the URL directly as the loadStrings() parameter - your sketch will then always use the latest data!
  • This file is tab - rather than comma - separated. You'll need to use split(textLines[i], "\t"); to split the text on the tab character.
    The \t represents a tab character in Processing.

Extra Challenge - Refactoring Your Visualization

If you have managed all of that, then try designing a DockingStation class that stores the name and other data that relate to the docking station. Refactor your code from the previous tasks so that data from each docking station is stored in a DockingStation object.

Hint: Make another tab and put the code that defines the DockingStation class in there. In your main sketch, you'll be able to create objects, store them in an array or ArrayList and display graphical representations of them.

Extra Challenge - Visualizing a Book

You may want to see whether you can visualize a text file. The methods you are using here with small data files can be applied to larger files too.

See what you can find at Project Gutenberg, elsewhere online, or in your own file-space. We will be impressed if you can find patterns in your data - and explain the code that enables you to do this data visualization.

See whether you can develop the sketches that you produced when we were considering The Time Machine to show where in the book the terms in a particular phrase occur? Or perhaps try this with key characters in a book you know well.
You could apply this approach to find words in a song, or characters in a screenplay.
Your solution might look something like this ...

This kind of sketch might enable you to find patterns in text files that you download, particularly if you add some interactions to change the phrases or characters that you are searching for.
For example, you may find data files online, and write sketches in class, that enable you to determine:

  • When do the terms 'good' and 'bad' appear in a text that you download from Project Gutenberg?
  • Which pairs of characters tend to appear together most commonly in the screenplay of Star Wars - A New Hope ?
  • Just how often did the The Beatles use the term 'yeah' in their lyrics and did this vary as time passed?

Consider a Book class that displays the content of a book in interesting ways and enables you to see summaries of the contents, and compare the 'data' in books.

Or, be inspired by Adam J Calhoun and his visualization of the Punctuation in Novels.
