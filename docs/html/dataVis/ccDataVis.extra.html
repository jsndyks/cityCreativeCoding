Bike Maps

Build on the Bike Map task in the Data Visualization : Core materials to add some interesting and useful interactive features to your sketch.

Challenge - Adding Interaction to the Cycling Data Scripts
Of course, there are many other things you can usefully do here.
See whether you can ...

  1. Show the differences between two times
    Hint: This will involve calculating differences between two floats that you extract from the array of values and you will end up with negative numbers - but we have thought about how to visualize negative numbers in the lecture.
  2. Only show stations that have particular names or geographic locations
    Hint: You'll have to record the station name as a String when you read values from the lines of text.
  3. Use squares or bars in the map instead of circles - you could use multiple bars to show a series of times concurrently;
  4. Scale the circles (or squares or bars) differently, to make them bigger or smaller;
Challenge - Adding MORE Interaction to the Cycling Data Scripts
Why not make things even MORE interactive?

  1. Can you add an interaction to the bar graph that allows us to scroll up and down through the data file, while always showing data for 25 stations?
    Hint: you will need to replace the numeric values used to identify the column in which the 9:00am data is stored, and the lines that are used in the loop, with variables that change in response to particular inputs.
  2. Can you scale the circles (or squares or bars) differently in response to key presses that make them bigger or smaller;
  3. Can you change the time that is shown dynamically, showing data for successive time periods?
    This could be according to some input (keyboard, mouse) or time (animation)
    You could do this for the graph or the map.
    Hint: You will need to read in one or more of the columns containing the number of bikes docked at the station. If you wish to show more than one time of day in the same sketch, you will probably need to store the bike numbers in a 2D array where 1 dimension is the docking station and the other is the time of day.
Extra Challenge - Live Bike Data Visualization

The bike scheme data that you have used so far is from 2015.

Live bike data, showing what is going on across London right now is available from https://www.staff.city.ac.uk/~sbbb717/tfl_bikes/latest-tsv.php.
Copy your solution from the previous task and modify it to use this live data.

Hints:

  • Open the latest data file in a browser to see the structure of the data.
  • You can put the URL directly as the loadStrings() parameter - your sketch will then always use the latest data!
  • This file is tab - rather than comma - separated. You'll need to use split(textLines[i], "\t"); to split the text on the tab character.
    The \t represents a tab character in Processing.
Extra Challenge - Refactoring Your Visualization

If you have managed all of that, then try designing a DockingStation class that stores the name and other data that relate to the docking station.
Refactor your code from the previous tasks so that data from each docking station is stored in a DockingStation object.

Hint: Make another tab and put the code that defines the DockingStation class in there.
In your main sketch, you'll be able to create objects, store them in an array or ArrayList and display graphical representations of them.

Book Graphics

You may prefer books to bikes, and so can have fun experimenting with visualization of literature.

Extra Challenge - Visualizing a Book

You may want to see whether you can visualize a text file. The methods you are using here with small data files can be applied to larger files too.

See what you can find at Project Gutenberg, elsewhere online, or in your own file-space. We will be impressed if you can find patterns in your data - and explain the code that enables you to do this data visualization.

See whether you can develop the sketches that you produced when we were considering The Time Machine to show where in the book the terms in a particular phrase occur? Or perhaps try this with key characters in a book you know well.
You could apply this approach to find words in a song, or characters in a screenplay.
Your solution might look something like this ...

This kind of sketch might enable you to find patterns in text files that you download, particularly if you add some interactions to change the phrases or characters that you are searching for.
For example, you may find data files online, and write sketches in class, that enable you to determine:

  • When do the terms 'good' and 'bad' appear in a text that you download from Project Gutenberg?
  • Which pairs of characters tend to appear together most commonly in the screenplay of Star Wars - A New Hope ?
  • Just how often did the The Beatles use the term 'yeah' in their lyrics and did this vary as time passed?

Consider a Book class that displays the content of a book in interesting ways and enables you to see summaries of the contents, and compare the 'data' in books.

Or, be inspired by Adam J Calhoun and his visualization of the Punctuation in Novels.
