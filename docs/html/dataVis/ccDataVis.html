1. Introduction to Data Visualization

So far, most of the sketches you have built have involved creating some form of visual representation that you as a programmer have controlled (e.g. by putting numbers into your program code or asking Processing to generate random numbers for you), or you as a user have influenced (e.g. by moving the mouse or pressing keys).

Data visualization is all about making meaningful, informative, readable, graphical representations of data. These data are usually created externally, outside your program, and you write code to read those data into your program and show them graphically. The data can be of all sorts of things - daily temperatures, Twitter messages, player performance in football matches, Moodle usage statistics, code commits, data generated in real time by sensors in the urban environment or complex model outputs that might, for example, be predicting patterns of disease.

Data can be turned into graphics in very different ways and for many different purposes. These range from

  • more artistic representations with more focus on the visual and interactive design than the data that underlie them;
  • more communicative representations where a particular story is being told using data; or
  • more scientific representations that help investigate scientific questions by assisting scientists in finding patterns in data and relationships between datasets.

Each of these will have different implications for visual design, but they all follow the same general pattern of transforming some form of dataset into some form of graphical depiction.

Examples

The web has many, many creative visual representations of data. This is in part due to the popularity and importance of Open Data and also because many of the various technologies that enable people to process this data - such as Processing. 2. Getting simple text data into a sketch

So how do we make a connection between the data we might want to visualize and our coding instructions? Thankfully Processing offers us several built-in methods for reading data, whether it comes from text files, activity trackers, remote databases or the Web.

Perhaps the easiest to use is the loadStrings() method that reads in a text file into an array of Strings.

To see how it works, lets adapt a sketch we have seen before that displays some names stored in an array: 

String[] names;

void setup()
{
  size(300,580);
  fill(0);
  textSize(14);  // -- try changing the size of the text

  names = loadStrings("names.txt");
}

void draw()
{
  background(255);
  float yPos = 25;  // -- try changing the starting position on the Y axis (25)

  for (String name : names)
  {
    text(name,20,yPos);
    yPos = yPos+28; // -- try changing the line by line vertical offset (28)
  }
}

Notice that nowhere in the code is there any list of the actual names to display. Instead the loadStrings() method looks for a data file called names.txt that contains the data - the text containing the names. Each line from this data file is read into a separate element in an array that we've called names in this example.

The draw() method then uses a for-each loop to iterate through this array, displaying each element one-by-one. So the output of our sketch is determined by the data. More names in the file, will give us more names on the canvas, with the same code.

In order for Processing to know where to look to find the file names.txt, it needs to be created somewhere and then dragged into the Processing window. This will make a copy of the file in the Processing sketch folder, from which Processing can access and read the lines of text.

An interactive example

Instead of a simple list of names, we can use the same approach to read in a larger text file. For example, Project Gutenberg provides a repository for free electronic copies of thousands of books. The content of all sorts of interesting books is available as text files. We can use this repository as a source of data for our sketches.

Below is a sketch that displays the contents of H.G. Wells' classic book 'The Time Machine' stored in a text file.

Each line of text in the file contains a whole paragraph, and the sketch reads the file in with loadStrings() and then displays one line at a time.

 

            String[] textLines;  // -- Lines of text will be loaded into this Array of Strings from file.
            int lineToDisplay;

            void setup()
            {
              size(600,400);
              // -- Use loadStrings() to load text into the textLines array from an online text file:
              textLines = loadStrings("https://jsndyks.github.io/cityCreativeCoding/data/theTimeMachine.txt");
              lineToDisplay = 142;    

              textSize(14);
              fill(0);
            }

            void draw()
            {
              background(255);
              text(textLines[lineToDisplay],5,5,width-10,height-10);
            }

The only element of this sketch that may be new is that we are using a slightly different version of Processing's text() method that takes 5 parameters ‚Äî the text string to display, the position of the top-left of the text area and the width and height of the text area. When specified like this (i.e. as a rectangle), Processing will attempt to lay out the text, line by line, within the rectangle. This saves us having to update the position of each line as we did in the first example.

Add this keyPressed() method to skip forward and backward a line with the < and > keys. This works by using the keys to update the variable lineToDisplay, which is used to determine the array index used to select the currently displayed line.

You can then change the line that is displayed interactively - use the < and > keys to display the book, one paragraph at a time.
Clicking the 'space bar' will display line 10 - this is the first line of the content of the book - the title.

                void keyPressed()
                {
                  if ((key == '<') && (lineToDisplay > 0))
                  {
                    lineToDisplay = lineToDisplay-1;
                  }
                  else if ((key == '>') && (lineToDisplay < textLines.length-1))
                  {
                    lineToDisplay = lineToDisplay+1;
                  }
                  else if (key == ' ')
                  {
                    // Line 10 is the first line of the book - it contains the title
                    lineToDisplay = 10;
                  }
                }

3. Extracting tokens from a line of text

The text that we have loaded so far is largely unstructured. While each line in the file is a sentence or paragraph of the book, and they are in the order in which they should be read, the text within the lines does not have any specific structure.

But data are often, and most usefully, stored in text text files with more explicit structure. Our early examples in names.txt were like this - data values separated by commas.

For example, each line may be a record about a thing that we are interested in, such as a GPS reading at a particular time, or a day of the week about which we have information. It may contain multiple pieces of information about the thing, such as the thing's position and elevation in the first case, or the thing's maximum heart rate recorded by an activity tracker for each hour of the day in the second. And if these pieces of information about multiple things are in a consistent order then we can compare the things (people, days, whatever).

We have seen that the loadStrings() method is very useful when you wish to read in lines of text into an array, line by line. But sometimes we may wish to go further than this and extract the individual elements within each line. For example, the words that make up a sentence or the numbers in the kinds of comma separated lists of numbers (or rows of numbers) in the elevation data or the heart rate data shown above. These elements are often referred to as tokens and the process of extracting them as parsing. This is what I did to extract the times and elevations from the elevationData.txt file to convert it into the time profile: I parsed each line of the file, and split it into tokens using the comma as a separator.

  • token - part of a longer String that relates to a particular data value;
  • separator - a character or String that separates tokens in longer Strings;
  • parsing - the process of extracting tokens from a String - usually involving splitting Strings at separators;

Processing provides a useful command to help with parsing tokens ‚Äî split().

This method will take a string and a separator as parameters and split the text up into an array of smaller strings, using the separator character to divide the text.

Read that sentence one more time!

This method will take a string and a separator as parameters and split the text up into an array of smaller strings, using the separator character to divide the text.

Let's say that again and take it slowly ...
If a String (such as one that you load from a line of a text file) contains a series of characters that are separated by a separator (such as a comma), then split() will convert the String into an array containing each of the parts of the String that are separated by the separator. These are the tokens!

Or as the Processing Reference Documentation puts it, split() ...

breaks a String into pieces using a character or string as the delimiter. The delim parameter specifies the character or characters that mark the boundaries between each piece. A String[] array is returned that contains each of the pieces.

So ...

  • String : "Sunday,55,51,49", which we can split with a ...
  • Separator : ",", and if we split the string by this separator we get ....
  • Tokens : "Sunday" "55" "51" "49"

If we were to use the split() method in Processing to split this String into tokens like this :
String tokens = split("Sunday,55,51,49",",");

Then our new array, tokens, would contain the following Strings... 

tokens[0] = "Sunday"                        
tokens[1] = "55"                        
tokens[2] = "51"                        
tokens[3] = "49"

Now we can read lines of text from a file and split each line up into arrays we can use the contents of files to inform our sketches. So we have an array of lines, and an array of tokens for each line that splits the line contents according to a particular separator.

Let's think about unstructured text first.
As an example, suppose we wished to display each of the words of a sentence in different colours: 

String textLine;

void setup()
{
    size(250,700);
    textSize(10);
    textLine = "The night came like the turning out of a lamp, and in another moment came tomorrow. The laboratory grew faint and hazy, then fainter and ever fainter. Tomorrow night came black, then day again, night again, day again, faster and faster still. An eddying murmur filled my ears, and a strange, dumb confusedness descended on my mind.";
}

void draw()
{
    background(255);
    float yPos = 18;

    String[] tokens = split(textLine," ");

    for (String token : tokens)
    {
        fill(random(50,200),random(50,200), random(50,200));
        text(token,10,yPos);
        yPos = yPos+12;
    }

    noLoop();    // Stops the draw() loop from repeating.
}

Here, the separator is the " " (i.e. a space character), which is what we normally use to separate words from one another in a sentence. The returned result after calling the method is an array of Strings, which in this example we have named as the array variable tokens. We then use a 'for-each' loop to iterate through that array, displaying each word in turn in a different random colour at a vertical position that increments by 12 pixels in each loop iteration - meaning that each successive token is 12 pixels below the previous one.

Extracting numeric values

The same principle can be used to extract numeric values from a line of structured text, like the ones we saw in the heart rate data above. As we have seen, numbers in a text file might be separated by commas rather than spaces, so it becomes a simple matter of passing a comma "," rather than a space " " as the argument used to define the separator when calling the split() method.

Here's an example that reads in a set of numbers representing daily temperatures and displays a series of rectangles that are sized according to the temperature of each day. Note that in this example (and the one above) we are assigning the value of the textLine variable explicitly within our code, but a real data visualization application is likely to read these values from a file using the approaches that we have learned about above üëÜ. 

String textLine;

void setup()
{
    size(400, 200);
    noStroke();
    fill(250, 110, 10);
    textLine = "14,16,18,19,19";    // Daily maximum temperatures in the next 5 days.
}

void draw()
{
    background(255);
    float xPos = 30;

    String[] tokens = split(textLine, ",");

    for (String token : tokens)
    {
        float temperature = float(token);    // Convert text into a number with float().
        rect(xPos, 100, 65, temperature);
        xPos = xPos + 70;
    }
}

The only command that may be new to you in this example is the use of a method called float() that converts the text of a number parsed from the String into a numeric float variable so that we can use it as a number and do some maths with it.

An obvious problem with the sketch is that the bar heights do not vary very much since we are representing the data (degrees Celsius) directly as pixel units (e.g. 19 degrees Celsius is shown as a rectangle 19 pixels high).

It is quite common in data visualization to want to rescale data values so they match the sizes in pixel units we wish to use. We could achieve this by multiplying each data value (stored in the temperature variable above) by some value, say 4, so that 19 Celsius becomes 19*4 = 76 pixels tall.

A more general and widely useful approach would be to use Processing's map() command that rescales any number from a range to the equivalent point on another range: 

String textLine;

void setup()
{
  size(400, 200);
  noStroke();
  fill(250, 110, 10);
  textLine = "14,16,18,19,19";    // Daily maximum temperatures in the next 5 days.
}

void draw()
{
  background(255);
  float xPos = 30;

  String[] tokens = split(textLine, ",");

  for (String token : tokens)
  {
    float temperature = float(token);    // Convert text into a number.
    float barHeight = map(temperature,0,30, 1,160);
    rect(xPos, 20, 65, barHeight);
    xPos = xPos + 70;
  }
}

The map() method requires 5 parameters to be specified.

The first is the number we wish to rescale. In the example above it is the value held in the temperature variable.

The next pair of parameters are the minimum and maximum data values we can expect our data value to fall between. This is the range that we map from or the source range. With UK temperatures, we will work on the assumption that our temperature range is between 0 and 30 degrees Celsius.

The final pair of numbers represent the minimum and maximum rescaled values. This is the range that we map to or the target range. In this case these values represent the number of pixels corresponding to the data minimum and data maximum respectively.

Map Basics:

map() is a method that returns a float, based upon 5 input arguments :
map(value, sourceRangeMin, sourceRangeMax, targetRangeMin, targetRangeMax)

Or maybe this is a better set or terms?

map(theNumberYouHave,
      theMinItCouldBe,
      theMaxItCouldBe,
      theMinOfTheRangeYouAreConvertingTo,
      theMaxOfTheRangeYouAreConvertingTo)

Copy the two lines of code that follow into a new new Processing sketch and vary the first three arguments to see how the value on the range sourceRangeMin to sourceRangeMax looks when 'mapped' to the target range from 0 to width. 

float x = map(10, 0,100, 0,width);
line(x,0, x,height);

So, if you scored 20/25 in a test and want to know what this works out as as a percentage, you could use:

float mark = map(20, 0,25, 0,100);
println("I scored 20/25, which is",mark,"%");

Or, if you have a data item that is 20/25 and want a bar that takes up 20/25 of the width of the screen then, you could use:

float barLength = map(20, 0,25, 0,width);
println("The bar needs to be 20/25 wide, which is",barLength,"pixels");

Our example attempts to visualize data from the London Cycle Hire Scheme.

Transport for London release some very large datasets that include the times and places of all journeys made using the bikes. But for now we are going to use a simpler dataset that records how many bikes are docked at a series of docking stations at various points in time. The first few rows and columns of this table of data look like this:

Each row (apart from the first 'header' row) contains data about a particular bicycle docking station. It has its own unique numeric identifier, or ID (column 0); a name (column 1); a geographic location specified as a pair of Ordnance Survey map coordinates (columns 2 and 3); the number of bikes capable of being docked at the station (column 4); and then the number of bikes actually docked at the station at 30 minute intervals from midnight to midnight (columns 5 to the end).

The full table can be downloaded as bikeShare.txt (right-click to download). Each bike station has its own line in the file and each value in every row is separated with a comma. This makes it an ideal format for reading with loadStrings() and then parsing with split().

Bike Bar Graph

The following sketch shows how we might read the data file, store the numbers in a series of arrays and use them to display how many bikes were docked at the first 25 stations in the table at 9:00am in the morning. 

// -- Bike Station Bar Graph Sketch
String[] stations;
int[]    availableBikes;
int[]    capacities;

void setup()
{
  size(600, 405);
  textAlign(LEFT,TOP);
  strokeWeight(0.5);

  String[] textLines = loadStrings("bikeShare.txt");

  // Create arrays for the data sized according to number of lines (we will ignore the header line - textLines[0]).
  stations       = new String[textLines.length];
  availableBikes = new int[textLines.length];
  capacities     = new int[textLines.length];

  // Iterate and store station name, capacity and number of bikes for each station: ignores header line
  for (int i=1; i<textLines.length; i=i+1)
  {
    // Split the line into 'values' array separated by a comma
    String[] values = split(textLines[i], ",");

    stations[i]       = values[1];        // -- column no. 1  in each line of the data file is the station name
    availableBikes[i] = int(values[23]); // -- column no. 23 in each line of the data file is 9am
    capacities[i]     = int(values[4]);  // -- column no. 4  in each line of the data file is station capacity
  }
}


void draw()
{
  background(255);
  float yPos= 2;

  // Show the first 25 stations with number of bikes at 9:00am: ignores header line
  for (int i=1; i<=25; i=i+1)
  {
    fill(0);
    text(stations[i], 5, yPos);

    noStroke();
    fill(50,50,200,100);
    rect(220,yPos+1,availableBikes[i]*8,14); // <-- available rect

    noFill();
    stroke(0);
    rect(220,yPos+1,capacities[i]*8,14);     // <-- capacities rect

    yPos += 16;
  }
}

Bike Map

Rather than visualizing this information as a series of bars, we could use the location data for each docking station to produce a simple map.

All we do here is plot a symbol for each station at a position on the canvas that is based upon the geographic information that we have read from the file - this is in columns 2 and 3.

For example, we might show each station as a circle. We could make the size of the circle relate to the number of bikes that the station is capable of storing (its capacity):

In the sketch that follows we read in values from columns 2 and 3 representing the coordinates of each docking station. These numbers are Ordnance Survey grid coordinates at 1m resolution.

To fit the data onto the screen we need to translate them into locations on the canvas in pixels. The map() command for rescaling the numbers will do this for us. But to get the scaling right, we need to know the geographic range of our data is. Here we store this in the variables NORTH, SOUTH, EAST and WEST. These are the geographic extents of our data - the boundaries of our map, the coordinates of the lines that define the top, bottom, left and right of the geographic area into which we are map()ping our data. 

// -- Bike Station Map Sketch
String[]  stations;
int[]     capacities;
PVector[] locations;     // <-- use a PVector to store station coordinates

float NORTH = 184600;    // Geographic coordinates defining the boundaries
float SOUTH = 177280;    //  of the area we are studying.
float EAST  = 539250;    // We will use map() to convert from the source range of
float WEST  = 523000;    //  geographic coordinates, to the target - our canvas

void setup()
{
  size(600, 405);

  String[] textLines = loadStrings("bikeShare.txt");

  // Create arrays for the data (ignoring header line).
  stations   = new String[textLines.length-1];
  capacities = new int[textLines.length-1];
  locations  = new PVector[textLines.length-1];

  // Iterate and store name, location and capacity for each station.
  for (int i=1; i<textLines.length; i=i+1)
  {
    // Split the line into values separated by a comma
    String[] values = split(textLines[i], ",");

    stations[i-1]   = values[1];
    capacities[i-1] = int(values[4]);
    locations[i-1]  = new PVector(int(values[2]),int(values[3])); // <-- coordinates are in cols 2 and 3.
  }
}

void draw()
{
  background(255);
  noStroke();
  fill(0,0,180, 70);

  for (int i=0; i<stations.length; i=i+1)
  {
    PVector location = locations[i];
    int capacity     = capacities[i];
    // Scale the locations (coordinates) to the canvas with map()
    float xPos = map(location.x, WEST, EAST, 10, width-10);
    float yPos = map(location.y, NORTH, SOUTH, 10, height-10);

    ellipse(xPos,yPos,capacity,capacity);
  }
}
