This session introduces the idea of loops to force the repetition of selected parts of your code.

By the end of this session you should be able to:

  • follow the flow of a program that loops;
  • create a for loop that repeats a block of code a fixed number of times;
  • recognise unnecessary repeated lines of code and replace them with a loop;
  • use nested loops to work with gridded structures

1. Program Flow and Loops

When coding you have come to rely on the fact that instructions are usually executed in the order you provide them in your code. Within a method such as setup(), draw() or the ones you create yourself, program flow moves from top to bottom one instruction at a time.

We have seen how we can modify this rule slightly by providing conditional blocks of code that are only executed if a particular condition is met. If it is not met, than the block of code is skipped. Even so, this still follows the general rule of code execution moving from 'top to bottom' within any one method.

We have also seen how we can move whole blocks of code into their own methods and call those methods at any time. This has allowed us to make repeated calls to a single block of code. Even though the use of methods makes program flow a little more complex, within any one method, we still follow the rule of moving 'downward' though each line of code from the top to the bottom of the method.

We can modify program flow further through the creation of loops. These are blocks of code that are repeated time after time — when program execution gets to the end of the loop it returns back to the start of the loop before repeating.

A loop is simply a block of code that is repeated a certain number of times. It will usually apply some form of test to check whether it should repeat or not. This allows the loop to be repeated a fixed number of times. Repeating some code in a programming language is known as iteration.

You have already taken advantage of one kind of loop — the draw() loop, which executes all the code inside draw() repeatedly, at a rate of 60 times every second.

In addition to repeating the whole of draw() we can make smaller blocks of code repeat by creating our own loops. These are very useful when we know we have to get the computer to do something many times and we want to save ourselves the bother of coding each instruction multiple times.

For example, suppose we wished to draw nine squares in a line:

One approach might be to issue the rect() command, once for each square. Something like this: 

    rect( 40,50,80,80);
    rect(140,50,80,80);
    rect(240,50,80,80);
    rect(340,50,80,80);
    rect(440,50,80,80);
    rect(540,50,80,80);
    rect(640,50,80,80);
    rect(740,60,80,80);
    rect(840,50,80,80);

As well as being a little tedious to type, repeating code like this makes it unnecessarily complicated to edit (e.g. suppose we wished to change the size of each square) as well as error-prone (did you spot the mistake in line 8?). Instead of writing the same commands or lines of code many times, loops allow us to write the code once, placing it inside a structure that requests the code be repeated many times.

Tip:
If you find yourself writing very similar code many times, the chances are that it can be improved in both efficiency and clarity by using a loop to bring out the regularity. One of the skills in programming is in being able to recognise that a particular task can be broken down into a repeated set of simpler tasks.

2. The for loop

One of the commonest forms of loops in Processing (and other languages) is the for loop. The example below shows how it might be used to draw those nine squares: 

for (int i=0; i<9; i=i+1)
{
  rect(40 + i*100, 50, 80,80);
}

A for loop has two parts — a header and a body. The body is the block of code between the curly braces: this is the code that will be repeated. The header is the line beginning with for and comprising three elements each separated by a semicolon. Those three elements are:

  1. A loop counter declaration and initialisation. In the example above it is the code int i=0;. This declaration is just like any other variable declaration in that we say what type of variable is being declared (the int type in this example) and we give it a name.

    It is common to give loop counters short names such as i, but you are free to give more explanatory names if you wish such as counter. That counter variable is only useable within the loop itself.

  2. A conditional test. This is the test that is carried out every time the loop repeats, and only if the result of the test is true will the body of the loop be executed. By allowing the result of the test to sometimes be false we can get the loop to repeat only a fixed number of times.

    In our example above the conditional test is i<9;. In other words, the loop code will only be executed if the loop counter i is less than 9.

  3. A loop counter assignment that applies some update to the loop counter. Typically this is to increase the value of the counter by one on each repeat. This is necessary otherwise the conditional test would never allow the loop to finish.

    In our example this element is i=i+1 but in principle it could be any form of assignment such as decreasing the counter by one or increasing it by 10.

The rules of variable declaration and scope also apply to loops. Variables declared in the header or within the braces of a loop are only usable within the loop itself.

A couple of other things to note when using for loops. The header line has no semicolon at the end. This is consistent with the coding of methods and conditional (if) blocks in that the semicolon is replaced with an open and close brace. It is only within the block of code inside the braces that the lines will have semicolons.

The code inside the braces of the loop should be indented. Again this is consistent with the way we have used indentation with braces when writing code in a method. Use CTRL-T (PC) or CMD-T (Mac) to make your indentation consistent - very useful, very important.

Notice also that in our example, we have used the loop counter variable i as part of the rect() command. This allows us to position each rectangle at a different x-coordinate every time the loop is repeated (0, 100, 200, 300 etc.). Most loops will use the counter to affect the instructions that are executed each time the loop draws.

Any code can go inside the body of a loop. For example, here is a modification to the woodland sketch we have previously developed, this time using a for loop to draw many trees. 

color leafColour,trunkColour;

void setup()
{
  size(800, 400);
  noStroke();
  leafColour  = color(82, 144, 97);
  trunkColour = color(95, 57, 26);
}

void draw()
{
  background(157, 238, 247);   // Sky
  fill(156, 245, 170);         // Ground.
  rect(0, height/2, width, height/2);

  for (float xPos=50; xPos<width; xPos=xPos+100)
  {
    drawTree(xPos,180);
  }
}

// Draws a single tree at the given position.
void drawTree(float treeX, float treeY)
{
  fill(trunkColour);
  rect(treeX, treeY, 20, 70);
  fill(leafColour);
  ellipse(treeX+10, treeY-30, 70, 90);
}

3. Nesting Loops

Let's return to a simple for loop that displays a row of shapes: 

for (int col=0; col<10; col = col+1)
{
  circle(20 + col*40,height/2, 30);
}

As we have seen, the loop allows us to code the circle() command only once, but for it to be called 10 times as the loop repeats.

We can think of all four lines of code comprising the entire loop as a single activity that draws a row of circles.

Just as we placed the circle() command inside a loop, we can do the same thing for our row drawing code.

In other words, we nest one loop inside another. This allows us to draw many rows of circles: 

  for (int row=0; row<10; row = row+1)
  {  // <-- commands issued in these braces happen 10 times 
  
    for (int col=0; col<10; col = col+1)
    { // <-- commands issued in these braces happen 100 times
        circle(20 + col*40, 20 + row*40, 30);
    }
  }

It is important to note that just because we have two loops in the code above doesn't mean we are only repeating something twice. Because we have placed the inner loop inside the body of the outer loop, the code() command is called 10 x 10 = 100 times.
Note how the indentation helps us to recognise this nested structure.

When you nest loops inside other loops or blocks of code, indentation becomes increasingly helpful as a way of making your code clearer to read. You can get Processing to apply that indentation automatically at any time by pressing Control-T (or Cmd-T on a Mac).

Nested loops are very useful when we deal with gridded structures and layouts.

5. Summary and Further Reading

  • While program flow is dominantly from top-to-bottom, we have seen how we can force flow to repeat sections of code using loops.
  • A for loop declares, initialises a loop counter, performs a conditional test on the counter to determine whether it should repeat and changes the value of the counter on each iteration of the loop.
  • Loops can be nested inside one another to provide a convenient way of handling gridded structures.
