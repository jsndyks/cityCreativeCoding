Complete the refactoring of the PathFinder sketch that we started in the lecture. Here's a reminder of what the draw() code should look like when the sketch is fully refactored: 

void draw()
{
  background(255);

  drawPeg(peg1X, peg1Y);
  drawPeg(peg2X, peg2Y);
  drawPeg(peg3X, peg3Y);

  if (inPlayMode)
  {
    moveBall();

    drawBall();

    bounceOffWalls();

    checkForPegCollision(peg1X, peg1Y);
    checkForPegCollision(peg2X, peg2Y);
    checkForPegCollision(peg3X, peg3Y);
  }
  else
  {
    drawBall();

    drawAimingLine();
  }
}
In the lecture we covered these methods:
  • drawAimingLine
  • drawBall
  • checkForPegCollision
In addition, you need to define the methods:
  • moveBall
  • bounceOffWalls
  • drawPeg
But remember, all the relevant code is already there in the sketch; you only need to restructure it.
DO NOT try to make all the changes in one massive edit. Make one change at a time. After each change, run the sketch to make sure that nothing has broken before moving on to the next change.

Extension exercises

Change the sketch so that the ball is set in motion by pressing the spacebar, rather than by pressing the mouse: remove the mousePressed() method, add a keyPressed() method.
While in aiming mode, allow the user to move pegs by clicking on them and dragging with the mouse. You will need to define three mouse event handlers: mousePressed(), mouseReleased(), mouseDragged(). You will need to add one or more new global variables to keep track of which peg is being moved during a drag operation.
