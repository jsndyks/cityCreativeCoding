These optional exercises give you practice at both using Processing's existing classes such as PVector, as well as creating your own. And plenty of fun making Robots.
Try them out if you completed the Objects : Core materials and feel like you need more of a challenge.
We don't expect many students to complete this activity, but some of you may enjoy the challenge and find it fun.

Make a Robot Class

... and a few Robot instances?
The code below provides a simple start point for creating a sketch that shows moving robots. It is based on the Ball class used in the lecture notes: 

Robot robot;

void setup()
{
  size(500, 500);
  robot = new Robot();
}

void draw()
{
  background(255);
  robot.draw();
  robot.move();
}

And in a new Tab for the Robot class ...

class Robot
{
  PVector position;   // Position of the robot.
  PVector velocity;   // Amount it moves on each redraw.
  float size;         // Size of robot.
  color colour;       // Colour of robot.

  // Initialises the robot's state.
  Robot()
  {
    position = new PVector(width/2, height/2);
    velocity = new PVector(random(-2,2),random(-2,2));
    size     = 30;
    colour   = color(0, 0, 0);
  }

  // Draws the robot at its current position.
  void draw()
  {
    noStroke();
    fill(colour);
    rect(position.x, position.y, size, size);
  }

  // Moves the robot according to its current velocity.
  void move()
  {
    position.x = position.x + velocity.x;
    position.y = position.y + velocity.y;

    if (position.x < 0)
    {
      velocity.x = -1 * velocity.x; // If robot at left of sketch move to right.
    }
    else if (position.x > width)
    {
      velocity.x = -1 * velocity.x; // If robot at right of sketch move to left.
    }

    if (position.y < 0)
    {
      velocity.y = -1 * velocity.y; // If robot at top of sketch move down.
    }
    else if (position.y > height)
    {
      velocity.y = -1 * velocity.y; // If robot at bottom of sketch move  up.
    }
  }
}

Copy the code provided above into a Processing sketch, placing the Robot class in a new Processing tab.

Modify the move() and draw() methods of the Robot class to make a more realistic looking robot that moves more like a robot. For example, instead of a single rectangle, you might show a set of shapes like the image shown.

Also, instead of moving diagonally, you might make the movement only up, down, left or right at any one time.

Note you should be able to complete this task by only modifying the code inside the Robot class. You shouldn't need to touch the code inside setup() or the main draw() methods.

Make Many Robots

Modify your robot sketch so that 50 robots move around the screen, not just one.

Note: you should be able to do this without making any further modifications to the Robot class.
All the necessary changes can be applied outside this class.
You will need to use Groups and Loops.
