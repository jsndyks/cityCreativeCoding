Image sequences for animation

We have seen that we can use multiple images on the canvas, with different processing and effects. But sometimes we would like to change the image that we show as time progresses. Displaying a series of images in sequence enables us to animate, and this is how we use images to produce movies (25 images per second) and games (canvas updates in relation to interaction with the screen).

For example, in a game, a character might have multiple images for different poses as they complete some action.

Arrays come in handy storing multiple images that represent a character in sequence. We can create an array of images, just as we have created arrays of integers, floating point numbers and Strings. Each image can be indexed by its position in the array as shown in the following code snippet.

      // -- Create an array of eight PImages called 'sprite'.
      PImage [] sprite = new PImage[8];

      // -- Use a loop to load a different image into each array element.
      for (int i = 0; i < 8; i=i+1) {
        sprite[i] = loadImage("man" + i + ".png");
        // -- Use println() for debugging and reporting :
        println("Loaded image\t","man" + i + ".png");
      }

      // -- Draw whichever image is in the element indexed by the variable 'frame'.
      image(sprite[frame], 0, 0);

This is just a few lines of code that may be helpful.
You'll need to decide which of these things need to go in setup() to be done once, draw() to be done repeatedly - frameRate times per second, and to be declared at the top of the sketch to ensure global scope.

Extra Challenge - Simple Game

Download the eight images and drop them into a sketch.

Then see whether you can animate them by building on the code snippet shown above.
You should try to :

  1. load the eight images into a PImage array
  2. draw an image at the mouse position with image
  3. make the image dependent upon the progress of the script with frameCount or millis()
       - try something like (millis()/100)%8
       - just println() this to the console at first if you don't see what it does
  4. vary the speed at which the animated runner moves through his running action
       - try varying the denominator that you use to divide millis()
  5. Move the runner along the screen as the sketch progresses

We can use animated graphics like this to develop characters in games and other applications.

For example, a basic side-scrolling game world consisting of a character running in a forest, might:

  1. Load images
  2. Draw the background, scrolling left (background image position moves a little to the left in each frame)
  3. Check input and update character position - jump?
  4. Draw the current sprite frame (the image of the character)
  5. Check for collisions
  6. End the game if a collision is detected
  7. Otherwise advance the frame
  8. Go to step 2

Extra Challenge - Charlie's Challenge

Download the Forest Run program.

  • Forest Run sketch and data

Make the game more playable by doing some of the following:

  • Using an array, add some scary obstacles along the path, rendered using images.
    Note: in the directory there is an image called "log.png" that you're welcome to use. The "camera" in the game is moving horizontally with the player. So when drawing objects, simply subtract the playerPosition.x from the location. For example, image(object, location[i].x -playerPosition.x, location[i].y); where location[i] is the location of the ith object.
  • If instead you'd prefer to use your own image instead of "stump.gif", but it lacks a transparent background, you can use image editing software (like LUNAPIC) to make the background transparent.
  • Implement a collision detection algorithm to test whether the player has collided with one of the obstacles.
    Note: included in the code is a method collisionDetection(), which can be used to see if two rectangular image objects are overlapping.
    Since the objects are not exactly rectangular, and there are extra pixels around the objects (particularly the sprite) in the images, some padding may be required to relax the collision detection.
  • Implement an appropriate action when the player collides with an obstacle
    (i.e., life lost or game over).
  • Using another array, add some pickups for the game.
    The data directory contains an image named "coin.jpg" that you can use for this purpose.
  • Apply collision detection to the pickups.
    If the player collides with a pickup, make it disappear (stop drawing it).
  • Implement a scoring system to keep track of the player's score.
  • Implement a heads up display that shows the player's score using the text() method.
  • Try other cool things - go for it!
