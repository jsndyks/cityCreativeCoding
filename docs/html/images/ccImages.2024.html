2. What is an image?

For our purposes, an image is a two-dimensional array, or regular grid, of colours. One can think of these as grid of miniature coloured rectangles positioned next to one another on the screen. They are much like a mosaic of tiles, each tile being of a single colour.

Mosaics are great - they allow us to create physical images and decorate our floors, bathrooms, window ledges ...

If you look carefully when you visit Paris, you'll see occasional tile mosaics made by Invader and various others who have followed this idea.

A Parisian Invader.

Jim Bachor's amazing pothole mosaics are gaining some traction too, though, with some exceptions, these tend not to use regular grids.

Literal Street Art, using mosaics,
by Jim Bachor.

To store regular grid mosaics digitally, we record information about the colour in each tile, and where each tile is positioned in the grid. The more tiles, the more detail.

We don't refer to these individually coloured elements of the image as tiles though. We refer to them as picture elements - or pixels for short.

And you will already have some ideas about how we specify colours digitally.

3.3 - Alpha... the 4th colour component

We can also specify the opacity of a colour - this is the extent to which is it not transparent. So far, we have only considered fully opaque pixels, that do not have any transparency. But we may want to create shapes or add images that overlap and have transparent properties. If we use the mosaic metaphor we can think of these tiles as coloured glass, and the opacity is the extent to which the glass does not let light through. Beer bottles tend to be very opaque, beer glasses tend to be much less so.

In a Processing colour definition we can add a fourth number to specify the opacity of a colour. This also ranges from :

  • 0 - fully transparent and so entirely invisible, to
  • 255 - completely opaque, the default if we omit the fourth argument to color()

Here are some examples:

              color darkBlue = color(0, 0, 100, 64);     // A dark blue that is 25% opaque.
              color darkRed  = color(100, 0, 0, 16);     // A dark red that is almost transparent.
              color myOrange = color(255, 128, 32, 255); // A lovely orange that is completely opaque.
              color anOrange = color(255, 128, 32);      // The same orange - alpha is 255 by default.

4.1 - Loading and displaying an image

Let's consider a simple example in which we load an image from a file (again) and display it on the canvas. We have done this before ðŸ‘†, so this should be a useful reminder.

The loadImage() method can load images into a sketch from various formats including GIF, TGA, JPG and PNG format. As we have seen, the input argument is the name of the image file. It's often best to load the image in the setup() method as we usually only need to do so once.

The image() method is what you use to actually draw the image, starting at an upper left pixel of the image (0, 0), at whatever position you give to image(). The first argument is a loaded image, the second and third are the coordinates of the canvas at which we want to draw the image.
If the image doesn't change in the sketch then we can often put the image() command in setup() too, so that both loading and drawing only happen once. 

PImage img;

void setup() {
  size(640, 480);
  img = loadImage("Holiday.png");
  image(img, 0, 0);
}

4.2 - Tint

A common application of image processing is to alter the colours across an image. In Processing, one can manipulate the colours of an image using tint(). If tint() receives one argument, only the brightness of the image is affected. Values range from 0 (no brightness) to 255 (full brightness).

So this will do nothing ...

tint(255);
image(img, 0, 0);

But this will darken the image. The lower the number, the darker night falls ...

tint(190);
image(img, 0, 0);

4.4 Image filtering

Much image processing involves using filters. Filters replace the colour values in each pixel of an image with new values, that either depend on the original colour value held in the pixel or those close to them.

Processing has powerful built-in support for image filtering using the filter() method.

For example, filter(INVERT) will invert the colours in an image, that is:

  • R = 255 â€“ R;
  • G = 255 â€“ G;
  • B = 255 â€“ B; 
PImage img;

void setup() {
  size(640, 480);
  img = loadImage("Holiday.png");
  image(img, 0, 0);
  filter(INVERT);
}

Methods such as filter(BLUR, N) will blur the image by replacing the colour of each pixel with a colour value based on the colours of pixels within a certain radius - N. A higher value of N results in more blurring.

Example: Variable blurring based on mouse location.

PImage img;

void setup() {
  size(640, 480);
  img = loadImage("Holiday.png");
}

void draw() {
  image(img, 0, 0);
  filter(BLUR, mouseX / 100);
}

4.5 - The Data : Accessing pixel and colour values

Individual pixel values can be accessed and changed using the get() and set() methods. We can get and set the pixels in an image and use these to select colours that we use in other parts of our sketch, for example colour other shapes.

Note that we never actually draw the image with image() in the following sketch, but as we have loaded it with loadImage() we can access its colours, which we can then use to fill() other shapes.
Specifically ... 

PImage img;
int s = 25;

void setup() {
  size(640, 480);
  img = loadImage("Holiday.png");
  stroke(64);
  rectMode(CENTER);
  background(127);
}

void draw() {                            // -- Get the colour of the PImage
  color c = img.get(mouseX, mouseY);   // -- called 'img' at cell mouseX,mouseY.
  fill(c);                              // -- Set fill to this colour.
  rect(mouseX, mouseY, s, s);          // -- Then draw a rectangle at
}                                        // -- the mouse position.

PImage img;
int s = 5;

void setup() {
  size(640, 480);
  img = loadImage("Holiday.png");
  rectMode(CENTER);
  background(127);
  noStroke();
}

void draw() {                            // -- Adds a bit of randomness!
  int rX = int(random(-25, 25));       // -- integer from -25 to 25 : x offset
  int rY = int(random(-25, 25));       // -- integer from -25 to 25 : y offset
  int mX = mouseX+rX;                   // -- Add some randomness around mouse X position.
  int mY = mouseY+rY;                   // -- Add some randomness around mouse Y position.
  color c = img.get(mX, mY);            // -- Get colour from 'img' at cell mX, mY.
  fill(c);                              // -- Set fill to this colour.
  rect(mX, mY, s, s);                   // -- Then draw coloured rect at mX,mY.
}

With added randomness we have a less precise, more spray paint alternative.

4.6 - The Data : Changing pixel and colour values

Just as we can get() a colour from a loaded image, we can use set() to change the colours being shown in the pixels on the canvas.
These may have been created by loading and drawing an image, or through other means.

Here's an example taken from the Processing Documentation that you can adapt:

for (int i = 30; i < width-15; i++) {
  for (int j = 20; j < height-25; j++) {
    color c = color(204-j, 153-i, 0);
    set(i, j, c);
  }
}
Try it out!
Copy the code into Processing - these two loops should give you a coloured canvas.
Adapt the code to change the colours on the canvas in different ways.

Copy, paste, run and adapt this code.
Use the Processing Reference Documentation to find out more.

This example shows how to set and change pixels in the display directly and dynamically.

void setup() {
  size(320, 240);
}

void draw() {
  for (int y = 0; y < height; y=y+1) {
    for (int x = 0; x < width; x=x+1) {
      float r = 255 * (x / (float) width);
      float g = 255 * (y / (float) height);
      float b = 127.5 + 127.5 * sin(millis() / 1000.0);
      set(x, y, color(r, g, b));
    }
  }
}
