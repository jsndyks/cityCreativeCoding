Once I have collected my data, can I present it on paper or do I have to do it digitally? If so what program shall I use

Errr ... we're asking you to use Processing to learn and demonstrate some competency in key concepts and constructs used in computer programming. So use Processing please.
We encourage you to sketch ideas on paper, and you may decide to take photos of some of these and use a PImage to incorporate them into your sketch. Be creative! But use Processing.

Can we submit sketches that aren't related to the data collection, alongside the data project? I.e just showing off our skills for fun!

We are all for showing off. We love showing off and encourage you all to do so too. We are sure that your personal tutors would like to see your sketches - call in during their office hours and show them what you can do. Blow them away!

BUT, for this exercise, we will be looking at up to 400 submissions, so we want to make this manageable. So please address the criteria (they are pretty straightforward) that we have specified and submit a single sketch.
Now, if you have done something fantastic that you want to share (who wouldn't?) we would encourage you to be creative and add some data to it. Make your fantastic show-off sketch a Dear Data sketch and then submit it. We can't wait!

Does the length of time data has been collected over matter

No.
Giorgia and Stefanie collected on a theme for a week in Dear Data, and you could do the same: or a month or a fortnight or a term or a day. Or you could compare two days or weeks or fortnights (but probably not terms).

The thing that does matter is that you have sufficient data to make your sketch interesting and revealing, and to learn something. So, if you are collecting for a short period, collect frequently. And in most cases you should collect plenty of data about the characteristics of the phenomenon that is your focus. This will give you a rich and potentially revealing data set. The Data Collection page provides some guidance.

If we present data in standard forms like in some form of chart or table that would generally be used to store that type of data, would that not be considered "creative"

Well "creative" is a nuanced concept.
But maybe it's worth asking yourself whether what you have done looks original, has original features, required you to do some thinking and problem solving to get it looking like you wanted it to - so that you could express yourself, find you voice.

If the answers are usually "No" then the answer to your question might be "Yes" - as in, not very creative.

Try SCAMPER or SCAPE!
Did you Substitute, Combine., Adapt, Put to Another Use, or Eliminate?
If not, then give one or two of those things a try and you may come up with something more original.

How would you reference your work without it being on the sketch?
Would you have to make a separate document with all the sources?

Make sure they are all listed in the sketch - perhaps use a "Sources" tab with the sources clearly listed in comments.

I still don't really understand how do we put the files onto GitHub. Do I just copy and paste all the code into a new GitHub folder?

In a nutshell: yes!
Simply go to your GitHub page and start a new repo from there. Clone the (empty) repo on your computer, and code away. Once happy with your changes/commits, push to the online repo. You will then see on your GitHub page all your code: to submit your work, share the link to this repo on the submission section on Moodle.

Please follow the "Intro to GitHub" Guide on Moodle on how to create/commit/push if need be.

What should I do in my spare time at home to become a better coder in java, especially if I've never learnt it before.

That's a question for the discussion area in the Java module.

But, having said that, we are using Processing because it is Java compatible and a great way in to the Java language. Working with classes, objects and methods as well as some of the constructs that we use to store and manipulate them (loops, arrays, array lists, etc.) will set you up nicely for Java.

So we recommend getting heavily into #CreativeCoding - that's the point really, Processing is a way in to Java, that's why we teach it, that's why we ask you to use it, that's why we put so much effort into the materials and the examples and into trying to give you the space, freedom and permission to experiment with it risk free in ways that are interesting and meaningful to you. We hope.
So ... get into and go crazy for #CreativeCoding.

Processing also helps with JavaScript - very useful way to start off.
