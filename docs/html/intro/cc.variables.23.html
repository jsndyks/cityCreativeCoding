This session introduces the idea of the variable — a named portion of the computer's memory used for storing data. Using variables is one way to make your programs more efficient, easier to read and adaptable.

By the end of this session you should be able to:

  • use Processing's built-in variables for storing mouse position and sketch size;
  • declare and initialise your own variables;
  • use println() to show the contents of a variable in the Processing console;
  • use variables for storing numbers, text and colours;
  • use appropriate variable scope in your coding design;
  • animate graphical objects in your sketch.

1. What is a variable?

In the previous session you were introduced to a moving rectangle sketch that drew a rectangle whose corner was at the current mouse position. We managed this by using two special variables provided by Processing — mouseX and mouseY. They are known as variables because their value can vary (as the mouse moves in this case). Even though the value of a variable can change, its name does not. At any point in our sketch code we can use the same name mouseX confident that it will contain the value of the current mouse position in pixel coordinates. This leads us to our first simple definition of a variable:

A variable is a named portion of a computer's memory that can store data. The data can be retrieved and updated within your program.

In the case of mouseX and mouseY the data value stored in each is a simple number (referring to the pixel coordinates of the current mouse position). Behind the scenes Processing keeps constant track of the position of your mouse and updates those variables with new numbers every time the position changes.

Copy the following code into Processing and try running it:

void setup()
{
  size(400,300);
}
 
void draw()
{
  println(mouseX);
}

The code above uses the new command println() to display the value stored in the variable mouseX. The value appears in Processing's 'console' (the black window below where you normally type in code). If you run the program and move the mouse left and right in the sketch you should see the number change as the mouse's horizontal x-position changes.

Processing has a number of built-in variables in addition to mouseX and mouseY. Of particular use are those called width and height, which store the width and height of the sketch (as determined by the numbers you provide with the size() command). These built-in variables are shown as pink when coding in the Processing window. We shall see why width and height are so useful a little later on. But for now, you can modify the code above to show all four of the variables we have considered so far:

void setup()
{
  size(400,300);
}
 
void draw()
{
  println(mouseX, mouseY, width, height);
}

If you were to run the program again, moving the mouse around the sketch window the console would display four numbers on each line corresponding to the mouse's x-position, y-position, the sketch's width and the sketch's height: 

87 118 400 300
255 117 400 300
298 160 400 300
310 195 400 300
318 236 400 300
281 239 400 300

Note how the last two numbers on each line do not change because the sketch's width and height remain unchanged as the mouse is moved.

Using println() can be helpful when you need to know the value stored inside a variable. This is especially true as your programs become more complicated and you need to debug them. And remember the output from println() doesn't appear in your sketch (unlike the rectangles and circles we have been drawing in previous sketches), but instead in Processing's coding console.

2. Declaring and initialising our own variables

We can, in addition to using Processing's built-in variables, create our own new variables. Creating a variable involves declaring a type, a name and a value. For the moment, will just consider variable types that can store numbers. These are referred to as float variables. We then just have to give our variable a name and a value. The name should always start with lower-case letters and not contain any spaces or special characters. For example,

float cost = 1.99;

Here we create a variable called cost and place the value 1.99 inside it. This line is doing two things — it is both declaring the variable's type and name (the type is float, the name is cost) and initialising its value (in this case with 1.99).

We could have chosen to do the declaration and the initialisation with two separate lines like this:

float cost;     // This is the variable declaration.
cost = 1.99;    // This is the variable initialisation.

Note the use of the use of the = symbol, which is referred to as the assignment operator. That means it takes whatever value is on the right of the = symbol and stores that value in the variable named on the left of the =.

Doing arithmetic on variables

When we assign a value to a variable, Processing will do any arithmetic you ask it to when calculating the value. For example

cost = 1.50 + 2.99;

would add 1.5 to 2.99 before storing the result (4.49) in the cost variable. This ability to do calculations is going to save you a lot of time as you develop more sophisticated programs. Other arithmetic operators you are likely to use include subtraction (-), multiplication (*) and division (/). These can be combined to do more complex sums. For example:

float snackPrice = 0.50;
float drinkPrice = 1.20;
float changeOwed;
 
float totalCost = snackPrice + drinkPrice;
float discount  = totalCost * 0.10;
 
changeOwed = 10.00 - (totalCost - discount);

Let's use our new ability to create our own variables to simplify the drawing of multiple shapes in a sketch:

Declaring and initialising a float variable

void setup()
{
  size(400,400);
  fill(0);        // Black fill colour.
}
 
void draw()
{
  background(255);
 
  float centreX = width / 2;
  float centreY = height / 2;
  float faceSize = 200;
  float earSize = 120;
 
  ellipse(centreX,centreY,faceSize,faceSize);
  ellipse(centreX-70,centreY-70,earSize,earSize);
  ellipse(centreX+70,centreY-70,earSize,earSize);
}

Notice how the variable centreX is assigned the value width/2. Processing will find the width of the sketch (which is 400 in this example), divide that number by two and then assign it to the variable centreX.

We can also apply similar types of arithmetic to the numbers provided to the ellipse() command. In the example above, we draw two 'ears' as smaller circles that are 70 pixels above the centre of the large 'face' circle and 70 pixels to the left and right respectively.

Why should we go to all the effort to declare and initialise these variables when we could have just put the number directly into the ellipse() command? The answer is that it makes it very easy for us to change parts of our program without having to recalculate all the new numbers that might result. For example try the following:

Three circles all using our own variables centreX, centreY, faceSize and earSize.

Copy the face sketch code above into Processing. Try changing the size of the sketch by replacing the 400s in the size() command to different values. Note how whatever you set the size to, the face is always drawn at the centre of the sketch. You can also change the value of faceSize and earSize to change the shape of the face.

3. Variable types

Remember that when we declare a variable we have to state what type of variable it is. So far, we have only considered one type —float. This is a useful type when you need to store numbers that might have a decimal point in them (such as our cost examples above). But Processing has many different types of variables, each suited to storing different types of data.

Here are some examples of other types that you may find useful.

The String type

If you wish to store some text, such as someone's name, you can store the letters that make up the text in a variable type called a String (note the use of the upper-case 'S'). String variables are declared and initialised just as we did for float types, but instead of assigning numbers, we assign text in quotation marks:

void setup()
{
  size(400,200);
  fill(0);
}
 
void draw()
{
  background(255);
  String welcomeMessage = "Hello from Bootcamp";
  textAlign(CENTER,CENTER);
  text(welcomeMessage,width/2,height/2);
}

Instead of providing the text "Hello from Bootcamp" directly to Processing's text() command, we assign it to a variable we have called welcomeMessage and then pass the contents of that variable to text(). Note also that we are doing some simple arithmetic to find width/2 and height/2 in order to position the text in the centre of the sketch.

The int type

This is another type of variable for storing numbers, but in the case of int variables, their values can only be whole numbers (i.e. without a decimal point). For example:

int numberOfCircles = 12;

int (short for 'integer') variables are commonly used when counting things. For example:

int counter = 0;
counter = counter + 1;

We will see some examples of using int variables to count in the next session.

The color type

Processing allows you to store a colour value in its own variable type called color (note the American spelling). This is useful if you wish to use the same colour in several places in your sketch. Here's an example:

void setup()
{
  size(400, 400);
  noStroke();
}
 
void draw()
{
  color skyColour    = color(157, 238, 247);
  color groundColour = color(156, 245, 170);
  color leafColour   = color(82, 144, 97);
  color trunkColour  = color(95, 57, 26);
 
  background(skyColour);
  fill(groundColour);
  rect(0, height/2, width, height/2);
 
  float treeX = 100;
  float treeY = 180;
 
  // Draw first tree
  fill(trunkColour);
  rect(treeX, treeY, 20, 70);
  fill(leafColour);
  ellipse(treeX+10, treeY-30, 70, 90);
 
  // Draw second tree to the right and down a bit
  treeX = treeX + 100;
  treeY = treeY + 30;
  fill(trunkColour);
  rect(treeX, treeY, 20, 70);
  fill(leafColour);
  ellipse(treeX+10, treeY-30, 70, 90);
 
  // Draw third tree to the right and up a bit
  treeX = treeX + 100;
  treeY = treeY - 30;
  fill(trunkColour);
  rect(treeX, treeY, 20, 70);
  fill(leafColour);
  ellipse(treeX+10, treeY-30, 70, 90);
}

The tree sketch above uses four color variables to store the colours of the sky, ground, tree leaves and trunk. The background(), and fill() commands then make use of the values in these variables without us having to type in the same RGB numbers each time we draw a tree. Note also how the code to draw each tree is exactly the same, the only thing that changes is the contents of the two variables treeX and treeY.

There are plenty of other variable types that we will come across later on, including types to store true/false values (boolean), single characters (char), images (PImage), fonts (PFont) and plenty of others. But for now, being able to store numbers, text and colours in variables should be sufficient.

Three trees using color variables to store trunk and leaf colours.

4. Variable scope

Remember that when using variables in Processing you do two things — you declare the variable, which is when you say what type of variable it is and give it a name; and you initialise it, which is when you assign a value to the variable. One of the reasons why it is important to separate declaration from initialisation is that where a variable is declared determines its scope.

The scope of a variable is the term given to where in your code that variable is recognised. The general rule is a variable can only be used in the same block of code in which it is declared. We can regard a 'block' as meaning within the pair of braces such as those that define a start and end of a method.

In practice this means that if you declare a variable inside, say, the setup() method, it could not be used inside the draw() method as each would be different blocks of code.

So would we ever need to use a variable in both the setup() and draw() methods? And if so, where should we declare such variables? The answer is yes we do sometimes need to use the same variables in several methods and to allow this to happen we simply declare the variable outside of all methods.

One of the reasons we might wish to declare a variable outside of the draw() method in particular is that it allows Processing to recall the value of the variable each time the method is called (remembering that draw() happens 60 times a second). Changing the value of a variable inside draw() is one way of creating simple animations. The following code draws some text at position (xPos,100) where xPos decreases its value by one every time it is redrawn:

float xPos;        // Declare xPos with sketch-wide scope.
 
void setup()
{
  size(600, 200);
  fill(0);
  textSize(48);
 
  xPos = width;    // Initialise xPos to right of sketch (600);
}
 
void draw()
{
  background(255);
  String message = "Hello again from the coding bootcamp...";
  text(message, xPos, height/2);
 
  xPos = xPos - 1;  // Decrease the value of xPos by one on each redraw
}

The variable xPos is declared as a float in the first line. This allows it to be used at any point in the sketch. It is initialised in the last line of setup() with the value of the width of the sketch (which is 600 in this case). In the draw() method some text is drawn using the current value of xPos (still 600 the first time it is drawn). The last line of draw() then decreases the value stored in xPos by 1. Remembering that draw() is repeatedly called 60 times a second, the text is then redrawn at position xPos which is now 599 before being reduced by one to 598 etc. The result is the text appears to scroll across the screen as xPos gradually decreases in value from 600 down to 0 and beyond.

Try to declare variables to be as 'local' as possible in scope. That is, if a variable is only needed in a single block of code, declare it inside the braces that define that block.

Here is another example that uses the same principle of declaring some variables outside the draw() method so we can animate by slowly changing their values in the draw() method. It is based on the Mickey Mouse sketch we created above, but this time, we will change the values of faceSize and earSize by 1% on each redraw:

// Sketch-wide variable declarations
float faceSize;
float earSize;
 
void setup()
{
  size(400, 400);
  fill(0);
  faceSize = 18;    // Variable initialisation.
  earSize = 12;
}
 
void draw()
{
  background(255);
  float centreX = width / 2;
  float centreY = height / 2;
 
  ellipse(centreX, centreY, faceSize, faceSize);
  ellipse(centreX-faceSize/3, centreY-faceSize/3, earSize, earSize);
  ellipse(centreX+faceSize/3, centreY-faceSize/3, earSize, earSize);
 
  faceSize = faceSize * 1.01;
  earSize = earSize * 1.01;
}

There is one more important implication to the fact that variables are bound to the block of code in which they are declared: a variable's identity is defined not just by its name but also by it's scope. Consider the following version of the scrolling text code:

void setup()
{
  size(600,200);
  fill(0);
  textSize(48);
 
  float xPos = width/2;    // Initialise xPos to right of sketch (300);
}
 
void draw()
{
  float xPos = width/2;
  background(255);
  String message = "Hello again from the coding bootcamp...";
  text(message,xPos,height/2);
 
  xPos = xPos - 1;  // Decrease the value of xPos by one on each redraw
}

If you copy and run this code in Processing you will notice that your text stays put. This is because the variable xPos in the setup method is not the same as the variable xPos in the draw method. They are two independent variables declared in different scopes that happen to have the same name.

So why is the text not moving? Our setup method declares and initializes a variable called xPos. However, as soon as our program exits the setup method, this variable ceases to exit.

Our program will then repeatedly run through the draw method. Every time our program enters draw, a new variable xPos is created and initialized. This variable is particular to that draw run and is discarded when the method exits. The change we make to this xPos is lost together with the variable itself. During the next draw iteration, a brand new instance of xPos will be created and initialized to the same constant value: width/2. Our text will thus be drawn in the same position over and over again.

Another problem occurs when two variables with the same name exist in overlapping scopes. This is called variable shadowing. Lets look at the following example:

float xPos = width; //Declare global xPos
 
void setup()
{
  size(600,200);
  fill(0);
  textSize(48);
 
  xPos = width;    // Initialise global xPos to right of sketch (600);
}
 
void draw()
{
  float xPos = width; // Notice: declare and initialise a local xPos
  background(255);
  String message = "Hello again from the coding bootcamp...";
  text(message,xPos,height/2);
 
  xPos = xPos - 1;  // Decrease the value of xPos by one on each redraw
}

Notice that here we declare xPos variables twice, once having global scope (at the top) and once in the draw method. Again, even though the variables have the same name they are different since they are defined in different blocks of code.

The problem occurs in the draw method where, given their scopes, both xPos variables should be visible. As such, which variable do the highlighted lines use? The rule is that in such cases the name identifies the variable with the more specific scope (in our case the xPos declared in draw). We say that xPos declared in draw shadows the global variable with the same name.


5. Summary and Further Reading

  • Processing contains a number of its own variables for storing the mouse position and sketch size.
  • You can create your own variables for storing numbers, text, colours and other features. This involves declaring the variable name and type and then assigning a value to the variable.
  • Variable names should be lower-case and should be given explanatory names.
  • The println() command can be used to display the contents of a variable, which can be useful when debugging a program.
  • You can use the draw() loop to incrementally change the contents of a variable to create an animation effect.
  • Variables are identified not just by their name but also by their scope. Make sure you work with the correct variable by considering not just its name but also its scope!
