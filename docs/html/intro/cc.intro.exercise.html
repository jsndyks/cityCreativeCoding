Before you attempt any of these exercises, make sure you completed all the coding and core tasks in the lecture notes!

These first exercises will give you practice at assembling commands inside Processing's draw() and setup() methods to draw collections of simple shapes. Attempt as many exercises as you can but on't worry if you don't manage to complete all these tasks within the lab session and remember to prioritise the core tasks.

Start with the moving rectangle sketch from the lecture notes (reproduced below). For this task it is OK to copy and paste the code directly into Processing (using the Edit Copy / Paste menus or keyboard shortcuts Ctrl-C and Ctrl-V). Run the sketch to check it works as expected.

// Sketch to draw a rectangle that follows the mouse.
void setup()
{
  size(400, 300);
  background(255, 236, 149);
  fill(203, 118, 122);
}
 
void draw()
{
  rect(mouseX, mouseY, 120, 80);
}

Try moving the background() line from setup() to the beginning of draw() (after the opening brace { but before the rect()line). What effect does this have on the sketch when you run it? Why is this? What happens when you move the fill() line to different parts of the sketch?

Create a new sketch that draws a simple rectangle that changes its size or colour depending on the position of the mouse.

Hint: you will have to use the special variables mouseX and mouseY instead of one or more of the numbers you would normally provide with a processing command.

Hint: if you use the mouse position to control colours, it may be convenient to size your sketch 256 x 256, so that the values of mouseX and mouseY are always in the range 0--255 (valid values for RGB colour components).
